# 桃桃 V2 北极星定位

## 1. V2 一句话定义

桃桃 V2 是两个人共同唤醒、共同确认、共同养成的 AI 小生命。

它不是把情侣功能接入 AI，也不是把聊天机器人包装成情侣助手。V2 的核心体验是：两个人把一个来自共同照片、宠物、礼物、玩偶、纪念物或默认胚胎的小东西唤醒成桃桃；桃桃拥有可成长的形象、性格、记忆、在场感和行动反馈，随后参与两个人的日常现场。

## 2. 必须避开的旧心智

V2 不能被用户理解成以下产品：

| 旧心智 | 为什么不成立 | V2 替代心智 |
| --- | --- | --- |
| 情侣空间 App | 会落到相册、纪念日、打卡、情侣任务。 | 两个人共同拥有一个正在醒来的小生命。 |
| 小恩爱/恋爱记升级版 | 会被看作传统情侣工具加更好看的 UI。 | 首次体验不是记录关系，而是唤醒关系中的第三个在场者。 |
| AI 情侣助手 | 会变成问答、建议、工具调用。 | 用户不是问 AI，而是在和桃桃一起生活。 |
| 关系分析器 | 会引导评理、诊断、谁对谁错。 | 桃桃只帮助缓和、表达、共同做小事，不裁判。 |
| 养成游戏 | 会让成长目标、任务、数值压过关系本身。 | 养成来自真实共同事件，不做每日压力。 |
| 会员/商城产品 | 会破坏小生命心智。 | 付费是正向时刻里的轻投喂和小窝变化。 |

## 3. 产品北极星

### 北极星体验

`两个人共同唤醒一个小东西 -> 双方确认它属于我们 -> 它在今日现场主动参与 -> 它记住共同发生的小事 -> 它在小窝里成长`

### 北极星指标

首发阶段建议使用：

`完成共同唤醒后 7 天内，至少发生 2 次双方参与的桃桃现场事件的关系数`

桃桃现场事件包括：

- 双方确认唤醒结果。
- 三方群聊中双方都出现。
- 今日现场完成一次轻行动。
- 一件小事被保存并由另一半确认。
- 小窝因共同事件产生可见变化。

不把单人浏览、单人聊天次数、照片上传量作为北极星。它们可以做漏斗指标，但不能代表 V2 是否成立。

## 4. 首次体验的产品命题

V2 首次体验不是注册、绑定、首页、功能入口，而是：

`唤醒你们的小东西`

第一屏必须让用户理解三件事：

1. 这里会出现一个只属于你们的小生命。
2. 它需要你和另一半一起确认才真正醒来。
3. 醒来后它不是头像或贴纸，而会进入你们的今日现场、群聊、记忆和小窝。

### 首次体验的推荐节奏

| 阶段 | 用户感受 | 产品任务 | 设计重点 |
| --- | --- | --- | --- |
| 看到胚胎 | 有一个小东西在等我们。 | 建立生命感和悬念。 | 极简、留白、微动效、低信息密度。 |
| 选择来源 | 它可以从我们的东西里醒来。 | 让用户上传或选择对象。 | 照片/宠物/纪念物/默认胚胎，不做素材库商店。 |
| AI 识别 | 它正在认出自己。 | 识别对象并生成候选。 | 过程可见，避免技术术语。 |
| 生成预览 | 它第一次睁眼。 | 展示动态形象、名字、性格。 | 强仪式感，少按钮，动作优先。 |
| 双方确认 | 它是不是我们的小东西。 | 建立共同拥有权。 | 明确谁已确认、谁待确认。 |
| 今日现场 | 它醒来后马上参与一件小事。 | 引导第一次双方共同行动。 | 不是功能宫格，只给一个主建议。 |

## 5. V2 的核心对象

### 5.1 桃桃不是资料卡，是生命对象

`TaotaoLife` 是 V2 的根对象。它包含：

- 形象：从来源对象生成，支持动态状态。
- 性格：由双方共同确认，可轻量调整。
- 记忆：来自共同事件和确认过的小事。
- 能量：不是惩罚数值，只代表最近是否被看见、被回应。
- 小窝：桃桃生活的空间，承接成长反馈。
- 在场：出现在首页、三方群聊、小程序回流、记忆详情和小窝。

### 5.2 情侣关系不是功能权限，是共同照料关系

`CoupleBond` 不只表示双方是否绑定，还表示：

- 谁发起唤醒。
- 谁确认了来源对象。
- 谁确认了桃桃形象。
- 哪些记忆属于共同确认。
- 哪些行为可以触发小窝成长。

## 6. 信息架构原则

V2 的信息架构不从功能分类出发，而从桃桃的生命状态出发。

推荐一级结构：

| 入口 | V2 含义 | 不应变成 |
| --- | --- | --- |
| 今日现场 | 桃桃醒来后今天想和你们做的一件事。 | 功能宫格、任务列表。 |
| 我们和桃桃 | 她、他、桃桃同时在场的三方群聊。 | AI 问答页、客服聊天。 |
| 记忆小窝 | 共同记忆和小窝成长合在一起的生命空间。 | 相册时间线、商城页。 |
| 桃桃 | 桃桃的样子、性格、状态、来源、成长记录。 | 设置中心。 |
| 邀请/确认 | 共同拥有权的关键状态。 | 登录流程附属步骤。 |

V2 可以继续保留 App + 小程序，但小程序只承接轻唤醒、邀请确认、今日现场回流，不复制完整 App。

## 7. 视觉与交互北极星

### 视觉关键词

- INS 极简。
- 一线科技 App 质感。
- 动态优先。
- 空气感、留白、浅层级。
- 真实内容和可见生命动作优先。
- 克制的材质感，不做低幼玩具堆叠。

### 交互关键词

- 一次只推进一个关键决定。
- 页面不解释功能，状态自己说话。
- 动效承担生命感，不靠文案喊“AI”。
- 按钮少但明确，主动作始终可见。
- 异常状态不打断生命感，要给可继续路径。

### 用户侧禁用表达

- AI 助手。
- 智能分析。
- 情侣工具箱。
- 恋爱任务。
- 打卡。
- 关系评分。
- 裁决。
- 谁对谁错。
- 机器人。
- 会员权益。

### 用户侧推荐表达

- 唤醒你们的小东西。
- 桃桃在认出自己。
- 你们要一起叫醒它吗。
- 桃桃第一次睁眼了。
- 等另一半也看一眼。
- 桃桃想把这件小事放进小窝。
- 今天桃桃先递给你们这一件事。

## 8. AI-native 的定义

V2 的 AI-native 不等于页面里有 AI 入口，而是：

1. 来源对象识别：从照片、宠物、玩偶、礼物、纪念物中识别可唤醒对象。
2. 生命形象生成：生成可动态表达的桃桃形象，而非静态头像。
3. 共同确认机制：AI 结果必须经过双方确认后进入共同关系。
4. 场景理解：桃桃能识别当前是唤醒、确认、今日行动、群聊、回忆还是安全风险。
5. 记忆沉淀：桃桃把被确认的小事转成轻记忆，不抓取、不过度总结。
6. 个性持续：名字、性格、说话方式、动作状态跨页面一致。
7. 安全前置：涉及攻击、控制、监控、自伤、暴力、羞辱时，先降温，不生成伤害内容。

用户不需要看见模型、Agent、Prompt、生成参数或技术状态。开发和设计需要把这些状态做成清晰的产品状态机。

## 9. 成功标准

V2 M0 成功不是“功能做全”，而是下面这些判断成立：

- 用户第一眼知道这是一个会被两个人共同唤醒的小生命。
- 用户不会把它误认为传统情侣 App。
- 一方发起后，另一方能理解为什么要确认。
- 桃桃醒来后马上进入一个真实今日现场，而不是停在生成结果页。
- 三方群聊让用户感到“我们和桃桃在一起”，不是“我在问 AI”。
- 记忆和小窝能证明桃桃在成长。
- 没有沉重功能，仍然足够惊艳、可上线、可继续迭代。
