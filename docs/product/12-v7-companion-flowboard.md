# 桃桃 V7.1 共同伴侣生产评审状态板

## 目标

V7.1 的交付物首先是生产评审状态板，不是一个只给人点击的 demo。状态板必须把真实 App 的每一步、每个角色视角、每个异常回收和每个研发字段讲清楚；状态板能过评审，才进入高保真 App 开发。

V7.1 修正两个核心错误：

- 正式 App 不能未登录先聊。主链路必须是登录/授权前置，再创建桃桃、邀请另一半、进入三人聊天。
- 桃桃不是工具机器人，也不是每句话都生成提议。桃桃首先是情侣共同 AI 伴侣，会聊天、玩梗、安静在场；只有明确落地意图才结构化。

## 状态板原则

- 手机预览里只出现用户能看到的话。
- 技术字段、意图、状态机、接口和埋点只能出现在状态卡或开发说明里。
- 每个状态必须回答：用户看到什么、下一步是什么、另一方看到什么、失败后怎么回收。
- 每个关键功能必须有发起方视角、接收方视角、完成态、取消/拒绝态。
- 情侣聊天不是任务页，不叫房间，不叫今日行动。
- 桃桃可接梗、可陪聊、可递话、可沉默；提醒和约定只是少数明确场景。

## V7.1 泳道

1. 注册登录前置：打开 App、手机号、验证码、个人称呼、异常、微信登录。
2. 创建桃桃：创建说明、上传照片、主体确认、不上传照片、生成中、生成失败、预览改名。
3. 邀请与绑定：生成邀请、发起方等待、小程序打开、接收方授权、接收方确认、绑定成功、晚点或拒绝。
4. 初次三人聊天：聊天初始、能力解释、第一次接梗、桃桃不插话、再次打开。
5. 日常陪聊：疲惫接话、共同梗、照片分享、只想吐槽、晚安、点名桃桃、补充照顾、少说一点。
6. 意图分流与升级门槛：默认只回应、好好说门槛、轻选择门槛、共同约定门槛、记忆门槛、安全优先。
7. 好好说：请求改写、编辑草稿、发送给对方、对方查看、不发送。
8. 轻选择与共同选择：单人轻选择、同步给对方、对方选择、选择结果、未回应。
9. 共同约定：发起草稿、发起方等待、接收方回应、对方轻改、双方同意、取消/拒绝、完成。
10. 提醒：单人提醒、共同提醒、通知权限、提醒触发。
11. 小窝记忆：建议留下、记忆草稿、对方确认、进入小窝、不留下、编辑删除。
12. 桃桃人格、边界和权限：桃桃页、改名、换来源、说话频率、隐私和解绑。
13. 异常、拒绝与安全：邀请过期、相册权限拒绝、网络失败、重复点击、对方拒绝、安全边界。
14. 研发交付检查：状态契约、文案边界、验收口径。

## 意图优先级

`safety_handoff` > `settings_boundary` > 当前事件动作 > `memory_capture` > `proposal_event` > `reminder` > `soften_message` > `decision_help` > `emotion_support` > `joke_play` > `companion_chat`

## 结构化门槛

| 输入类型 | 是否结构化 | 说明 |
| --- | --- | --- |
| “今天电量只剩 3%” | 否 | 情绪陪伴，桃桃接话即可 |
| “你是杯子精吗？” | 否 | 玩梗，不建任务 |
| “发了一张晚霞照片” | 否 | 照片分享，不自动沉淀 |
| “晚上吃什么” | 否 | 先给轻选择 |
| “给阿川三个选项” | 是 | 需要对方参与 |
| “今晚 20:30 去公园，问问阿川” | 是 | 时间、行动、对方点头齐全 |
| “明早提醒我带伞” | 是 | 明确单人提醒 |
| “这件事想留下” | 是 | 记忆草稿，双方确认 |
| “桃桃少说一点” | 是 | 设置边界，不是聊天事件 |

## 验收标准

- `/?mode=flowboard` 显示 `data-prototype-version="v7.1"`。
- 状态板至少包含 14 条泳道、110 个状态画板。
- 不得出现“未登录先聊一句”作为正式 App 主链路。
- 手机预览不得出现 `intent=`、`event.status`、`structuredCard`、`data-` 等内部语言。
- 手机预览不得出现“AI 心灵信号”“今日行动”“房间”等错误产品口径。
- 每个状态画板必须包含手机预览、下一步、评审检查项、开发要点。
- 每个状态必须具备可排期字段：owner、phase、priority、apiContract、analyticsKey、dependencies。
- 必须包含足够多的日常陪聊非结构化状态，证明桃桃不是事件生成器。
- 必须包含双方视角、拒绝/取消/过期/网络失败/权限失败/安全边界。
