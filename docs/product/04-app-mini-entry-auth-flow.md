# App + 小程序入口与授权流

## 1. 设计原则

注册登录是 V2 生产版的安全入口。登录后的第一业务体验，才是“有个小东西可以被你们唤醒”。

授权不是中途补丁，而是上传照片、调用生成、创建邀请、确认关系、写入记忆和跨端同步之前的状态边界。

目标：

- App 里先用最短路径完成身份识别和会话恢复。
- 小程序里用微信授权承接邀请和轻确认。
- 两个入口共享同一套 `CoupleBond`、`AwakenSession`、`TaotaoLife` 状态。
- 不做复杂账号中心，不让用户在注册页理解完整产品。

## 2. App 入口流

```text
打开 App
-> 手机号一键登录/验证码
-> 创建或恢复 UserProfile
-> 轻资料
-> 展示 seed 态小东西
-> 用户选择照片/默认小物
-> 创建 AwakenSession
-> 生成桃桃候选
-> 发给另一半确认
```

### App 登录触发点

| 触发点 | 是否需要登录 | 说明 |
| --- | --- | --- |
| 打开 App | 是 | 先创建或恢复 `UserProfile`，避免匿名滥用上传和生成资源。 |
| 看 seed 态 | 是 | seed 是登录后的第一业务状态。 |
| 选择默认小物 | 是 | 进入 `AwakenSession` 前必须有用户身份。 |
| 上传照片 | 是 | 上传前必须登录，避免未授权内容进入处理链路。 |
| 生成候选 | 是 | 需要绑定 `AwakenSession`。 |
| 发给另一半 | 是 | 需要 `CoupleBond` 草稿和邀请 token。 |
| 今日现场 | 是 | 需要已唤醒或已绑定状态。 |

### App 登录 UI 要求

- 默认使用轻页面，不跳到营销式注册页。
- 主按钮只写当前任务，例如 `发送验证码`、`继续`、`保存并进入桃桃`。
- 手机号输入、验证码输入、失败重试、协议确认必须有状态。
- 登录成功后进入唤醒流程，不进入账号中心。

## 3. 小程序入口流

```text
微信分享卡/轻链接
-> 打开小程序
-> 读取 invite token
-> 微信授权
-> 展示来源对象和桃桃候选
-> 接受 / 想改一下 / 不是这个
-> 状态写回 App
-> 打开 App 或继续一次轻行动
```

### 小程序授权触发点

| 触发点 | 是否需要微信授权 | 说明 |
| --- | --- | --- |
| 打开分享卡 | 否 | 可展示轻预览和发起方。 |
| 查看完整候选 | 是 | 需要识别用户，避免确认错人。 |
| 接受绑定 | 是 | 需要写入 `CoupleBond`。 |
| 轻行动选择 | 是 | 需要同步到 `TodayScene` 或 `TriadMessage`。 |
| 回 App | 否 | 可带 token，App 内再恢复会话。 |

### 小程序页面状态

| 状态 | 用户看到 | 主动作 |
| --- | --- | --- |
| `mini_invite_preview` | 小雨邀请你见见桃桃。 | 微信授权继续 |
| `mini_auth_failed` | 授权失败说明，不丢失邀请。 | 重新授权 |
| `mini_review` | 来源对象、桃桃候选、发起方。 | 我也确认 |
| `mini_confirmed` | 双方确认完成。 | 打开 App |
| `mini_light_action` | 今天的一件轻行动。 | 先用一次桃桃 |
| `mini_open_app_failed` | 无法打开 App。 | 复制链接 |

## 4. 共享状态

| 状态对象 | App 写入 | 小程序写入 |
| --- | --- | --- |
| `UserProfile` | 手机号登录、恢复会话 | 微信授权创建或绑定 |
| `CoupleBond` | 创建邀请、绑定成功 | 接受/拒绝邀请 |
| `AwakenSession` | 上传、识别、生成、发起确认 | 确认已有候选 |
| `TaotaoLife` | 生成、命名、性格、醒来 | 只读或确认 |
| `TodayScene` | 生成主行动、完成行动 | 轻选择写回 |
| `TriadMessage` | 完整聊天 | 轻行动回流卡 |
| `MiniLinkSession` | 创建 token、openApp payload | 授权、确认、回流 |

## 5. 边界

- 小程序不做完整图片生成。
- 小程序不做完整注册表单。
- App 生产版必须先登录再上传、生成和邀请。
- 未授权用户不能确认共同关系、写入共同记忆或访问双方历史。
- 任一端失败都必须保留原任务上下文。

## 6. M0 验收用例

1. 未登录用户打开 App，只能先完成手机号登录/验证码/轻资料。
2. App 登录成功后进入唤醒流程第一步。
3. A 在 App 发出邀请，B 从小程序打开后能看到同一个来源对象。
4. B 微信授权失败时，邀请 token 不丢失。
5. B 确认后，A 的 App 能进入桃桃醒来状态。
6. B 在小程序完成一次轻行动后，App 的我们和桃桃里出现回流卡。
7. 小程序无法打开 App 时，能复制轻链接。
