# 桃桃 V2 核心流程与状态机

## 1. 总体流程

V2 的主流程以“生命状态”推进，而不是以功能入口推进。

```text
进入 -> 手机号注册/登录 -> 识别当前关系 -> 发起唤醒/加入唤醒 -> AI 识别来源对象 -> 生成桃桃候选 -> 双方确认 -> 桃桃醒来 -> 今日现场 -> 三方群聊 -> 记忆小窝 -> 桃桃持续成长
```

App 承载完整体验；小程序承载轻入口、邀请确认、轻现场和回 App。

## 2. 全局业务对象

| 对象 | 说明 | 关键字段 |
| --- | --- | --- |
| `UserProfile` | 单个用户身份。 | `userId`、`nickname`、`avatarUrl`、`phone/WxOpenId`、`platformBindings` |
| `CoupleBond` | 双方关系和共同拥有权。 | `coupleId`、`memberA`、`memberB`、`bondStatus`、`createdBy`、`confirmedBy` |
| `AwakenSession` | 一次唤醒会话。 | `sessionId`、`coupleId`、`initiatorId`、`sourceType`、`sourceMediaId`、`recognitionStatus`、`generationStatus`、`confirmStatus` |
| `TaotaoLife` | 桃桃生命对象。 | `taotaoId`、`coupleId`、`lifeStatus`、`name`、`sourceObject`、`persona`、`motionSet`、`currentMood`、`energyState` |
| `TodayScene` | 今日现场的一次主行动。 | `sceneId`、`type`、`participants`、`inputSummary`、`outputSummary`、`status` |
| `TriadMessage` | 她、他、桃桃三方消息。 | `messageId`、`senderType`、`senderId`、`contentType`、`sceneContext`、`safetyStatus` |
| `NestMemory` | 记忆小窝中的共同小事。 | `memoryId`、`sourceSceneId`、`title`、`mediaIds`、`taotaoNote`、`confirmStatus`、`nestEffect` |
| `MiniLinkSession` | 微信轻入口和邀请确认会话。 | `miniSessionId`、`sourceChannel`、`targetAction`、`token`、`status`、`openAppPayload` |

## 3. 全局状态机

### 3.1 关系状态 `bondStatus`

| 状态 | 含义 | 可见能力 | 下一步 |
| --- | --- | --- | --- |
| `guest` | 未登录或未授权。 | 只能登录/授权，不能上传、生成或创建邀请。 | 登录 App 或微信授权。 |
| `single_ready` | 已识别当前用户，未建立关系。 | 发起唤醒草稿、邀请另一半。 | 创建 `CoupleBond` 草稿。 |
| `invite_created` | 一方已创建邀请，未分享或未打开。 | 邀请卡、复制/微信分享。 | 对方打开。 |
| `invite_opened` | 对方打开邀请，未确认。 | 邀请来源、发起人、轻预览。 | 接受或拒绝。 |
| `pending_pair_confirm` | 双方身份已识别，等待共同确认。 | 确认关系，不开放完整共同体验。 | 绑定成功。 |
| `bound` | 双方已绑定。 | 完整唤醒、今日现场、群聊、记忆、小窝。 | 唤醒或继续使用。 |
| `unbind_pending` | 解除绑定确认中。 | 冷静确认、影响说明。 | 取消或解除。 |
| `unbound_archived` | 关系解除，历史按规则归档。 | 个人可见边界、重新开始。 | 新建关系。 |

### 3.2 桃桃生命状态 `lifeStatus`

| 状态 | 含义 | 产品表现 |
| --- | --- | --- |
| `seed` | 有一个小东西在等你们，未选择来源。 | 极简胚胎、微动效、主按钮“唤醒它”。 |
| `source_selected` | 已选择照片/默认来源。 | 显示来源卡和可编辑入口。 |
| `recognizing` | AI 正在识别可唤醒对象。 | 进度、对象轮廓、高级但不技术化的动态。 |
| `object_confirm` | 识别出候选对象，等待用户选中。 | 候选对象、换图、使用默认形象。 |
| `generating` | 正在生成桃桃候选。 | 生成进度、可取消、失败降级。 |
| `preview` | 候选形象可预览。 | 动态预览、名字、性格、动作。 |
| `pending_partner_confirm` | 一方确认，等待另一半。 | 谁已确认、谁待确认、提醒对方。 |
| `awake` | 双方确认，桃桃正式醒来。 | 第一次睁眼、进入今日现场。 |
| `active` | 桃桃已进入日常循环。 | 今日现场、群聊、记忆小窝。 |
| `resting` | 暂时无新事件或用户选择少打扰。 | 安静陪伴，不催促。 |
| `needs_attention` | 有待确认事件或异常需要处理。 | 只提示一件最重要的事。 |

## 4. App 完整流程

### 4.1 首次进入

```text
打开 App
-> 手机号/验证码或已有会话
-> 识别 bondStatus
-> 未绑定：发起唤醒草稿 + 邀请另一半
-> 已绑定未唤醒：进入唤醒流程
-> 已绑定已唤醒：进入今日现场
```

首屏规则：

- 不先展示传统首页。
- 不铺情侣功能。
- 未登录不能上传照片、调用生成、创建邀请或进入共同体验。
- 登录是安全边界，不是中途弹窗；登录完成后再进入唤醒桃桃。
- 登录页不做复杂账号中心，只完成手机号、验证码和轻资料。

### 4.2 首次唤醒

```text
auth
-> seed
-> 选择来源：照片 / 宠物 / 礼物 / 玩偶 / 纪念物 / 默认胚胎
-> 上传或选择媒体
-> AI 识别候选对象
-> 用户确认对象
-> 生成桃桃候选
-> 预览动态形象
-> 设置名字、性格、说话方式
-> 发给另一半确认
-> 双方确认
-> 桃桃醒来
-> 今日现场第一件事
```

设计要求：

- 每一步只保留一个主动作。
- 识别和生成过程要可见，不能只转圈。
- 不出现模型名、AI 参数、风格词堆叠。
- 用户可随时换图、使用默认胚胎、稍后继续。
- 生成结果不直接进入共同空间，必须双方确认。

### 4.3 AI 识别

输入：

- 用户上传照片或相册选择。
- 来源类型标签。
- 用户手动框选或确认对象。

输出：

- `sourceObjectCandidates`：候选对象。
- `confidence`：只给开发使用，不在用户端展示。
- `safetyStatus`：是否可生成。
- `fallbackReason`：识别失败、敏感内容、对象太模糊、照片不可用。

用户侧状态：

| 状态 | 文案方向 | 可操作 |
| --- | --- | --- |
| 识别中 | 桃桃在认出自己。 | 取消、稍后继续。 |
| 多对象 | 你想让哪个小东西醒来。 | 选一个、重新上传。 |
| 低置信 | 桃桃有点看不清。 | 手动圈一下、换图、默认胚胎。 |
| 不可用 | 这张图不适合唤醒。 | 换图、默认胚胎。 |

### 4.4 生成

生成目标不是“生成头像”，而是创建 `TaotaoLife` 初始形态。

生成结果至少包含：

- 主形象。
- 头像裁切。
- 待机动作。
- 开心动作。
- 思考动作。
- 第一次睁眼动作。
- 小窝站位图。
- 基础性格摘要。

M0 可用 2.5D/短动效实现；不要求实时 3D。

### 4.5 双方确认

```text
发起方确认
-> 生成确认卡
-> 另一半打开 App 或小程序
-> 看到来源、形象、名字、性格
-> 接受 / 请求调整 / 拒绝
-> 双方接受后 lifeStatus = awake
```

确认卡必须展示：

- 来源对象。
- 桃桃预览。
- 名字和性格。
- 发起方。
- 当前确认进度。
- 接受、想改一下、不是这个。

确认规则：

- 一方可以发起并预览。
- 另一方不确认时，不能进入完整共同体验。
- 另一方请求调整后，回到预览编辑，不重新丢失来源。
- 拒绝后保留发起方草稿，可换图或默认胚胎。

## 5. 今日现场

今日现场是 V2 醒来后的第一主屏，不是功能首页。

### 5.1 首屏结构

今日现场首屏必须像一个正在发生的生活片刻，而不是功能首页。

1. 今日照片现场：占首屏主要视觉，优先使用餐桌、房间、宠物、纪念物等真实生活图像。
2. 桃桃动态在场：桃桃出现在现场里，像从生活物件里醒来，不像独立贴纸或客服头像。
3. 今日决定卡：只解释今天这一件事，压缩为一句关系理解和一个可执行动作。
4. 一个主行动：进入三方群聊、轻决定、轻表达、共同确认之一。
5. 一个轻切换控件：允许换一件事，但不能把多个能力同权摊开。

禁止结构：

- 不展示她/他/桃桃三条信号列表。
- 不展示吃什么、好好说、记小事等同权入口。
- 不把今日现场做成任务中心、功能卡片流或内容 feed。

主行动由状态机决定：

| 条件 | 主行动 |
| --- | --- |
| 刚完成唤醒 | 和桃桃说第一句话。 |
| 双方都在线或刚确认 | 让桃桃递出今天第一件小事。 |
| 有待确认记忆 | 先确认这件小事。 |
| 晚饭时段 | 一起决定吃什么。 |
| 有轻冲突语境 | 先把话说软一点。 |
| 无明显上下文 | 桃桃给一个低压力小建议。 |

### 5.2 今日现场行动状态

| 状态 | 含义 |
| --- | --- |
| `suggested` | 桃桃递出主行动。 |
| `started` | 一方开始。 |
| `waiting_partner` | 等另一半参与或确认。 |
| `completed` | 双方完成。 |
| `memory_prompted` | 可转成小窝记忆。 |
| `nest_updated` | 小窝有变化。 |
| `dismissed` | 用户跳过，不惩罚。 |

## 6. 三方群聊：我们和桃桃

### 6.1 入口

三方群聊的产品名建议使用 `我们和桃桃`，它表达她、他、桃桃同时在场。

入口来源：

- 今日现场。
- 桃桃醒来完成页。
- 小程序轻回流。
- 记忆详情。
- 小窝变化。
- 系统提醒。

### 6.2 消息类型

| 类型 | 说明 |
| --- | --- |
| `user_text` | 她或他说的话。 |
| `taotao_text` | 桃桃回应。 |
| `action_card` | 桃桃递出的轻行动。 |
| `confirm_card` | 需要双方确认的卡片。 |
| `memory_card` | 可保存或已保存的小事。 |
| `nest_update` | 小窝变化。 |
| `safety_card` | 降温或安全拦截。 |
| `mini_return_card` | 小程序轻选择回流。 |

### 6.3 场景识别

用户输入进入：

```text
消息 -> 安全初筛 -> 场景识别 -> 取关系/桃桃/记忆上下文 -> 生成候选 -> 桃桃口吻合成 -> 安全复检 -> 输出
```

可识别场景：

- 普通陪聊。
- 今天做什么。
- 吃什么。
- 好好说。
- 约会灵感。
- 记一件小事。
- 确认回忆。
- 小窝反馈。
- 争执降温。
- 高风险安全。

### 6.4 群聊边界

- 桃桃不站边。
- 桃桃不替用户监控另一半。
- 桃桃不读未授权聊天记录。
- 桃桃不诱导付费。
- 桃桃不在争执中触发投喂。
- 桃桃不把私密单人输入默认变成共同记忆。

## 7. 记忆小窝

V2 建议把“共同回忆”和“小窝成长”合并成 `记忆小窝` 的产品心智。

### 7.1 记忆生成流程

```text
今日现场/三方群聊/小程序轻选择
-> 桃桃识别可沉淀的小事
-> 生成记忆草稿
-> 创建者确认
-> 另一半确认
-> 写入记忆小窝
-> 小窝产生可见变化
```

记忆草稿字段：

- 标题。
- 时间。
- 来源现场。
- 文字。
- 图片或截图。
- 桃桃一句话。
- 可见范围。
- 待确认方。

### 7.2 小窝变化

小窝变化必须和真实事件相关：

| 事件 | 小窝反馈 |
| --- | --- |
| 第一次唤醒 | 出现桃桃初始位置。 |
| 第一次三方聊天 | 小窝亮起第一盏灯。 |
| 晚饭决定完成 | 桌上出现小餐牌或暖光。 |
| 好好说被发送 | 桃桃状态变柔和。 |
| 记忆双方确认 | 墙上出现一张小照片。 |
| 投喂成功 | 出现小布丁、礼物或感谢纸条。 |

### 7.3 记忆状态

| 状态 | 说明 |
| --- | --- |
| `draft` | 创建者可编辑。 |
| `pending_partner` | 等另一半确认。 |
| `confirmed` | 进入共同记忆小窝。 |
| `needs_edit` | 另一半请求调整。 |
| `rejected` | 不进入共同空间，可保留个人草稿。 |
| `hidden` | 共同隐藏或权限变更。 |
| `deleted` | 删除后保留必要审计记录。 |

## 8. 我们和桃桃

`我们和桃桃` 不是后台设置页，而是关系、桃桃、边界的轻管理区。

首屏信息：

- 两个人。
- 桃桃当前状态。
- 已确认的共同记忆数。
- 最近小窝变化。
- 桃桃来源和名字。
- 待确认事项。

二级能力：

- 桃桃的样子：名字、性格、说话方式、来源照片、动作。
- 我们的关系：昵称、纪念日、邀请状态、解绑。
- 共同边界：照片授权、记忆确认、通知、数据删除。
- 少打扰一点：桃桃提醒频率和安静模式。

状态要求：

- 改名后全局同步。
- 删除来源照片后保留已生成形象，除非用户选择删除形象。
- 解除关系必须说明共同记忆、小窝和桃桃的处理方式。

## 9. 微信轻入口与邀请确认

小程序不是完整 App 镜像。它只解决微信里的轻进入、轻确认、轻回流。

### 9.1 小程序状态

| 状态 | 入口 | 展示 | 主动作 |
| --- | --- | --- | --- |
| `wx_guest` | 未授权。 | 桃桃 seed 卡。 | 微信授权。 |
| `mini_single` | 授权后未绑定。 | 发起人的唤醒邀请或默认小窝卡。 | 接受/创建邀请。 |
| `invite_review` | 从分享卡进入。 | 来源对象、桃桃预览、发起方。 | 我也确认。 |
| `mini_bound_seed` | 已绑定但未唤醒。 | 轻唤醒入口。 | 打开 App 继续。 |
| `mini_bound_awake` | 已绑定已唤醒。 | 桃桃轻状态、今日一个动作。 | 轻选择或回 App。 |
| `mini_pending` | 有待确认事项。 | 唤醒/记忆/行动确认卡。 | 确认或稍后。 |

### 9.2 小程序可做

- 微信授权。
- 打开邀请。
- 接受或拒绝绑定。
- 查看唤醒确认卡。
- 确认桃桃候选。
- 做一个轻选择，例如今晚吃什么方向。
- 把轻选择写回三方群聊。
- 查看一条小窝变化。
- 打开 App 继续完整体验。

### 9.3 小程序不做

- 完整图片生成流程。
- 深层小窝装修。
- 完整记忆时间线。
- 完整我们页设置。
- 大量聊天历史。
- 复杂支付闭环。
- 多层筛选或任务系统。

## 10. 异常状态

异常状态必须有可见 UI，不只用 toast。

### 10.1 账号与关系异常

| 异常 | 处理 |
| --- | --- |
| 手机号登录失败 | 保留输入，给重试和换方式。 |
| 微信授权失败 | 说明授权用途，允许稍后再试。 |
| 邀请过期 | 展示发起人和过期原因，可重新发送。 |
| 邀请已被他人使用 | 阻断并提示联系发起人。 |
| 当前用户已有关系 | 展示已有关系，不能直接覆盖。 |
| 双方重复创建邀请 | 合并提示或选择保留一个，不让用户丢失草稿。 |

### 10.2 唤醒异常

| 异常 | 处理 |
| --- | --- |
| 上传失败 | 重试、换图、使用默认胚胎。 |
| 相册权限拒绝 | 说明用途，给系统设置和默认胚胎。 |
| 识别失败 | 支持手动圈选、换图、默认胚胎。 |
| 识别出多个对象 | 让用户选一个。 |
| 图片敏感或不适合 | 不生成，给温和说明和替代路径。 |
| 生成超时 | 保留会话，通知稍后回来，不丢草稿。 |
| 生成结果不满意 | 轻量重生成或调整性格，不重新走全部流程。 |
| 另一半拒绝 | 回到预览编辑，保留来源和理由。 |

### 10.3 今日现场异常

| 异常 | 处理 |
| --- | --- |
| 另一半未响应 | 显示等待，不惩罚；允许先放入草稿。 |
| 行动生成失败 | 重试、换一个轻行动、手动记录。 |
| 用户跳过 | 进入安静态，不连续催促。 |
| 弱网 | 本地保留输入和选择，恢复后同步。 |

### 10.4 群聊与安全异常

| 异常 | 处理 |
| --- | --- |
| 消息发送失败 | 原地重试，不丢输入。 |
| 桃桃回复失败 | 显示桃桃没想好，可重试或换个说法。 |
| 攻击/羞辱/威胁 | 显示降温卡，不生成伤害内容。 |
| 监控/定位/控制请求 | 明确拒绝，并给健康替代动作。 |
| 自伤/暴力风险 | 降级为安全提示，必要时建议现实求助。 |

### 10.5 记忆与小窝异常

| 异常 | 处理 |
| --- | --- |
| 记忆保存失败 | 保留草稿，允许稍后重试。 |
| 图片上传失败 | 删除坏图、重传、不影响文字保存。 |
| 另一半不确认 | 记忆留在待确认，不进入小窝。 |
| 小窝资产加载失败 | 展示稳定占位和重试，不白屏。 |
| 投喂支付失败 | 不改变小窝状态，允许重试或取消。 |
| 负面情绪场景投喂 | 不展示投喂，桃桃先安抚。 |

## 11. 事件埋点

M0 必须埋以下事件，便于判断 V2 是否成立：

| 事件 | 触发 |
| --- | --- |
| `seed_view` | 首次看到小东西。 |
| `awaken_start` | 点击唤醒。 |
| `source_selected` | 选择来源。 |
| `recognition_success/fail` | 识别成功或失败。 |
| `generation_success/fail` | 生成成功或失败。 |
| `preview_confirm_self` | 发起方确认预览。 |
| `preview_confirm_partner` | 另一半确认。 |
| `taotao_awake` | 双方确认完成。 |
| `today_scene_start` | 今日现场开始。 |
| `today_scene_complete` | 今日现场完成。 |
| `triad_chat_both_participated` | 三方群聊双方都出现。 |
| `memory_partner_confirmed` | 记忆被另一半确认。 |
| `nest_changed` | 小窝发生变化。 |
| `mini_confirm_complete` | 小程序确认完成。 |
