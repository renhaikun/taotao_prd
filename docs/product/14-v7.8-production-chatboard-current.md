# 桃桃 V7.8 生产评审状态板当前说明

更新日期：2026-06-22

## 一句话定义

桃桃是情侣双方和 AI 伙伴共同相处的三人聊天 App。产品不把 AI 做成工具箱，也不把情侣关系经营做成任务流，而是在聊天中由桃桃作为第三个参与者自然介入，并在需要时生成可回收、可修改、可确认的小事。

## 当前交付范围

- 10 条生产流程
- 7 层聊天覆盖矩阵
- 106 个手机状态稿
- 112 张当前截图证据
- Vite React 可运行高保真原型
- Playwright E2E 验收脚本

## 运行入口

```bash
cd prototype
npm install
npm run dev -- --port 5175
```

生产评审状态板：

```text
http://127.0.0.1:5175/?mode=chatboard
```

## V7.8 核心流程

### 1. 登录前置

用户必须先完成手机号登录和验证码验证。未登录状态不允许进入聊天输入。

登录流程只保留必要动作：

- 手机号登录
- 验证码登录
- 验证码错误轻提示

不进入注册流程的内容：

- 上传照片
- 生成 3D 桃桃
- 命名桃桃
- 邀请另一半

这些都后置到产品使用过程中。

### 2. 未绑定可使用

手机号登录成功后，如果还没有绑定另一半，用户仍然进入可用 App。

此时关系栏只展示：

- 当前用户
- 桃桃

顶部胶囊显示：

- 邀请另一半

用户可以先和桃桃聊天，不强制立刻邀请。

### 3. 邀请另一半

邀请入口只来自顶部胶囊。点击顶部胶囊后出现邀请半屏。

邀请半屏有两个明确分支：

- 微信分享
- 手机号搜索

这两个分支不能随机出现，必须都来自邀请半屏里的显式选择。

### 4. 微信分享路径

流程：

1. 用户点击顶部邀请胶囊。
2. 半屏出现“微信分享”和“手机号搜索”。
3. 用户点击“微信分享”。
4. 进入邀请卡预览。
5. 用户点击微信发送或复制链接。
6. 发送完成后，聊天流不新增任何已发送气泡。
7. 顶部胶囊变成“等对方加入”。
8. 对方打开分享卡。
9. 对方手机号登录并确认加入。
10. 进入三人聊天。

重点规则：

- 分享卡预览是用户主动选择后的结果。
- 发送成功只改变顶部胶囊状态。
- 不允许在聊天流里出现“我发过去了”之类流程播报。

### 5. 手机号搜索路径

流程：

1. 用户点击顶部邀请胶囊。
2. 半屏出现“微信分享”和“手机号搜索”。
3. 用户点击“手机号搜索”。
4. 进入手机号搜索半屏。
5. 用户输入或选择对方手机号。
6. 用户点击发送绑定提示。
7. 发送完成后，聊天流不新增任何已发送气泡。
8. 顶部胶囊变成“等对方加入”。
9. 对方在 App 或小程序收到绑定提示。
10. 对方确认绑定或选择不是本人。
11. 确认后进入三人聊天。

重点规则：

- 手机号绑定必须由接收方确认，避免误绑定。
- 发送提示成功只改变顶部胶囊状态。
- 不允许在聊天流里出现“手机号也发了”之类流程播报。

### 6. 三人聊天底座

绑定成功后，关系栏变成三人头像：

- 小雨
- 阿川
- 桃桃

顶部胶囊切换为“今天的小事”。

聊天流只保留：

- 小雨消息
- 阿川消息
- 桃桃消息
- 桃桃消息下的小能力胶囊
- 必要的轻系统行

不允许出现：

- 独立大卡乱飞
- 内部状态文案
- 技术说明
- 工具箱式功能堆叠

## 顶部胶囊规则

顶部胶囊是索引和找回机制，不是聊天内容。

优先级：

1. 邀请另一半
2. 等我看
3. 已保存
4. 对方看过
5. 已定
6. 今天的小事

发送失败留在原消息现场，不进入顶部胶囊优先级。

## 消息内能力胶囊规则

桃桃要先作为第三个人说话，然后能力调用附着在桃桃自己的气泡下方。

示例：

1. 用户说“那你给个做法。”
2. 桃桃回一句自然语言。
3. 桃桃气泡下出现小胶囊。
4. 生成完成后，小胶囊变为可打开状态。
5. 用户点击小胶囊后才打开详情半窗。

不允许：

- 用户说完一句话后直接弹半屏
- 把生成状态放成大卡片挡住聊天
- 把 AI 能力当系统工作流播报

## 详情半窗规则

详情半窗必须来自用户主动点击。

详情半窗中，修改逻辑应当是：

1. 用户输入想怎么改。
2. 发送输入即代表让桃桃修改。
3. 原消息下的小胶囊进入生成或更新状态。
4. 生成完成后，用户打开看到更新后的内容。

不再设置“发送后下面再来一个让桃桃改”的重复按钮。

## 验收脚本

核心脚本：

- `prototype/scripts/e2e-v72-chatboard-acceptance.mjs`

重点覆盖：

- 未登录不能聊天。
- 验证码组件正常和错误状态尺寸一致。
- 默认桃桃就绪不作为可见注册步骤。
- 未绑定用户能进入可用聊天。
- 邀请路径从顶部胶囊半屏显式分叉。
- 微信发送后不新增聊天气泡，只改变顶部胶囊。
- 手机号提示发送后不新增聊天气泡，只改变顶部胶囊。
- 对方确认绑定前必须有确认页。
- 手机预览中不出现内部状态、技术字段和解释性文案。
- 三人聊天关系栏渲染头像组。
- 普通消息发送失败使用气泡旁重试按钮。

## 下一步研发建议

1. 冻结 V7.8 对象状态机和顶部胶囊优先级。
2. 把邀请绑定、三人聊天、今天的小事抽屉拆成接口契约。
3. 先做 App 端聊天主界面，再做小程序接收确认轻端。
4. 将桃桃人格回复和能力调用拆成 agent 层策略，不在 UI 层硬编码。
5. 对登录、绑定、消息发送失败、生成失败、误触发和安全边界建立端到端验收。
