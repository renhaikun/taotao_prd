# 桃桃 V7.5 聊天主界面生产状态板

## 目标

V7.5 聚焦“进入聊天前置门槛 + 聊天主界面”。目标是把“手机号登录、伴侣绑定、桃桃创建、情侣双方 + 桃桃三人聊天、气泡尾注式能力胶囊、顶部索引胶囊、可继续输入修改的详情半窗、私下整理半窗、共同小事确认、异常回收”打磨成可以进入生产评审的界面状态板：用户能看懂，研发能排期，产品能检查每个状态的触发、收起、唤醒、可见性和异常回收。

这版修正 V7.2 之后仍然存在的核心问题：

- 所有聊天之前必须先完成登录；未登录、未绑定、未创建桃桃都不能伪装成真实三人聊天。
- 聊天不是工作流。桃桃首先像第三个在场的人一样聊天、接梗、陪伴、沉默。
- 消息内能力胶囊是 AI 能力调用的第一现场。桃桃先像第三个人一样接话，再把生成中、完成、失败、可打开的能力入口挂在自己的消息下方。
- 消息内能力胶囊只做气泡尾注，不解释完整内容。小胶囊只保留最短状态，完整说明放到详情半窗。
- 顶部胶囊常驻。它不是主状态，也不把消息内胶囊收走，只做全局索引、快捷找回和私下找桃桃入口。
- 可修改详情半窗必须支持继续和桃桃说。用户输入修改方向后，原消息内胶囊进入修改中、已更新或失败状态。
- 消息流里只出现人说话、桃桃消息下的能力胶囊和必要的轻入口。桃桃的内容必须是头像 + 气泡，不允许用系统卡片伪装成聊天。
- 群聊里说出的请求，桃桃回应和整理结果默认双方可见。不能把公开说出的话偷偷变成单侧草稿。
- 私下创建事务必须来自明确入口，例如输入区加号、桃桃头像、胶囊旁入口或未发送输入中的“找桃桃整理”。
- 半屏层只由用户主动触发。用户点消息内能力胶囊、点顶部胶囊、显式私下创建事务、发送前编辑或安全边界时才出现。
- 技术语言只能在评审文档和测试属性里，不能出现在用户手机预览里。

## 生产对象契约

胶囊不是主状态。研发实现时，主状态落在对象上，胶囊只是把当前最重要的对象投影到聊天顶部。

| 字段 | 用途 | 关键枚举 |
| --- | --- | --- |
| `objectType` | 这是什么对象 | `chat_message`、`taotao_reply`、`result_card`、`proposal_card`、`reminder_card`、`personal_draft`、`confirmation_item`、`counter_item`、`shared_memory`、`exception_item` |
| `lifecycleStatus` | 对象走到哪一步 | `auth_required`、`partner_not_bound`、`source_missing`、`generating`、`previewing`、`private_draft`、`delivered`、`viewed`、`pending_partner_confirm`、`pending_initiator_ack`、`confirmed`、`rejected`、`failed`、`archived` |
| `visibilityScope` | 谁看得到 | `group_visible`、`private_owner_only`、`partner_visible_unhandled`、`shared_editable`、`shared_readonly` |
| `participantStatus` | 双方行为进度 | `not_visible`、`visible_unread`、`viewed`、`accepted`、`countered`、`rejected`、`deferred`、`visible` |
| `capsule` | 顶部投影样式 | `none`、`idle`、`rotating`、`active`、`quiet`、`summary`、`error` |

## 主界面五区

1. 关系栏：稳定展示小雨、阿川、桃桃的三人关系，以及桃桃的轻状态。
2. 顶部索引胶囊：轮询显示当前最重要的状态，右侧 `+` 可新建事务；点击后优先定位原消息现场。
3. 消息流：只放三个人的气泡和桃桃消息下的能力胶囊。没有推荐区、独立结果卡、内部状态卡。
4. 输入区：默认鼓励继续说话。更多入口只做附件/表达入口，不默认展开功能面板。
5. 半屏层：临时处理层。可收起，收起后回到聊天，不进入消息历史。

## 详情半窗规则

| 层级 | 作用 |
| --- | --- |
| 结果内容层 | 展示当前版本，例如做法步骤、记忆文字、提醒内容或待确认事项 |
| 快捷修改 | 给出高频修改方向，减少输入成本 |
| 继续输入 | 用户可以自然语言告诉桃桃想怎么改 |
| 动作层 | 保存、发送、确认、取消、重试 |

用户输入修改后，不直接新增大卡片。原消息内胶囊更新为 `修改中`，完成后变为 `已更新`，用户点开后看到新版内容并可继续改、保存或发送。

## 可见性边界

| 场景 | 可见性 | 正确行为 |
| --- | --- | --- |
| 小雨在群聊说“今晚吃什么” | 群聊可见 | 阿川、桃桃都能看到，桃桃用气泡自然接话 |
| 小雨在群聊说“桃桃帮我整理一下” | 群聊可见 | 桃桃先用气泡回应，完整整理挂在桃桃消息下的能力胶囊里 |
| 群聊里双方同意“收成几个选择” | 共享事项 | 桃桃消息下出现轻入口，顶部胶囊提供快捷找回 |
| 小雨从输入区加号或桃桃头像进入整理 | 私下整理 | 阿川当前看不到，半屏必须明确提示 |
| 私下整理后点“发到聊天里” | 共享事项 | 群聊出现来源说明，桃桃消息下挂共享入口，顶部胶囊同步索引 |
| 用户已在群聊说出的内容 | 群聊可见 | 不能回收成私有草稿，只能公开接话或公开整理 |
| 用户从胶囊 + 进入 | 私下整理 | 半窗里可推荐小事、和桃桃私聊、保存个人草稿 |
| 用户退出未完成半窗 | 私下整理 | 必须提供先留着、继续整理、不保存 |
| 发起方发给对方后 | 对方可见未处理 | 发起方看到未打开、已看未决定、晚点、拒绝、正在修改等分支 |

## 胶囊规则

| 情况 | 胶囊显示 | 原因 |
| --- | --- | --- |
| 纯聊天、玩梗、情绪陪伴 | 常驻空/轮询胶囊 | 不打扰聊天，但保留新建入口 |
| 群聊整理结果 | 消息内能力胶囊 + 顶部索引 | 整理结果留在原消息现场，顶部只负责找回 |
| 私下整理、好好说草稿 | 常驻空胶囊 | 对方还看不到，不能显示私有内容 |
| 无需对方同意的小事 | 低存在感胶囊 | 直接进入发起方胶囊，不向群聊发卡片 |
| 需要对方同意的小事 | 消息内入口 + 顶部索引 | 入口给对方处理，处理后消息现场保留，顶部同步变轻 |
| 接收方点过入口 | 胶囊持续显示 | 消息内入口变轻，顶部索引仍可找回 |
| 胶囊详情 | 半屏混合列表 | 同时展示已同意、待同意、无需同意 |
| 对方轻改 | 消息内入口 + 顶部索引 | 修改发回群聊，由发起方点开确认或继续修改 |
| 已定 | 低存在感胶囊 | 事情已经完成主要处理，只保留轻锚点 |
| 失败或权限异常 | 消息内失败胶囊 + 顶部索引 | 失败保留在原消息现场，顶部提供找回 |

## 生产验收口径

- `/?mode=chatboard` 根节点必须是 `data-prototype-version="v7.6"` 与 `data-surface="chatboard"`。
- 默认入口仍是 V6，`?mode=flowboard` 仍是 V7.1，小程序入口优先级最高。
- 页面必须展示 7 层覆盖矩阵，且每层引用的状态 id 都必须在状态板中真实存在。
- 每个状态必须有手机预览、关系栏、消息流、输入区、六个评审槽位。
- 前置门槛状态必须没有胶囊、没有输入区、不能出现聊天消息。
- 聊天状态必须有常驻胶囊和输入区。
- 桃桃在消息流中必须是头像 + 气泡。
- 每个状态必须声明可见性：`group`、`private` 或 `shared`。
- 每个状态都必须声明对象类型、生命周期、可见范围和参与者状态。
- 聊天状态都必须显示一个顶部索引胶囊，且胶囊右侧必须有私下整理入口。
- 能力调用必须先挂在桃桃消息下方，呈现生成中、完成、失败、已打开等消息内能力胶囊。
- 消息内能力胶囊必须位于桃桃消息内部，不能作为脱离上下文的独立大卡片。
- 消息内能力胶囊必须是轻量尾注，不承载长标题、完整说明或多个操作。
- 可修改详情半窗必须有结果预览、快捷修改和继续输入框。
- 修改类状态必须覆盖修改中和已更新两个回流状态。
- 群聊可见状态不得出现“对方看不到”的私有草稿提示。
- 私下整理状态必须来自明确私下入口，并在半屏里提示另一方当前不可见。
- 默认结果、整理、修改、提醒和异常都不能用独立大卡片压住聊天；需要处理时用消息内能力胶囊承接。
- 半屏层只在 `overlay=active` 的状态出现，并必须有可执行按钮和来源：`message-capability`、`top-capsule`、`explicit-private-entry`、`message-long-press` 或 `safety-boundary`。
- 手机预览不得出现 `intent=`、`event.status`、`structuredCard`、`data-`、`AI 意图分流`、`今日行动`、`房间`、`工作流`、`任务`、`待处理`、`权限失败`、`异常胶囊`、`AI 推荐`、`处理中`、`状态` 等内部语言或错误口径。
