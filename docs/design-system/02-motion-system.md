# 桃桃 V2 Motion System

## 0. Motion-first 原则

V2 的桃桃不是靠大角色图和功能卡存在，而是靠运动方式存在。

Motion thesis：

> 动态要像真实生活里的轻微注意力变化：呼吸、看见、靠近、等待、记住、把一句话长成下一步。

运动目标：

- 让用户感到桃桃在场，但不抢走真实照片和聊天内容。
- 让 AI 过程可感知，但不炫技。
- 让状态变化有连续性：看见 -> 想一想 -> 给出行动 -> 等待确认 -> 记住。
- 保持一线消费级 App 的克制，不做游戏化表演。

## 1. 动效语言

### 1.1 节奏

| 类型 | 时长 | 用途 |
| --- | --- | --- |
| Micro | 80-160ms | pressed、icon state、轻反馈 |
| Short | 180-320ms | sheet、toast、局部按钮、识别标记 |
| Medium | 360-700ms | 行动卡生长、照片切换、小窝物件出现 |
| Long | 900-1800ms | 唤醒生成、桃桃 idle loop、邀请等待 |

规则：

- 默认使用 180-320ms，不把所有反馈做慢。
- 动作生长可以慢一点，但用户操作反馈必须快。
- loading 至少 400ms，避免闪烁。
- 连续动画之间需要 80-120ms 呼吸间隔。

### 1.2 曲线

| 曲线 | 建议 | 用途 |
| --- | --- | --- |
| Standard | cubic-bezier(0.2, 0, 0, 1) | 页面元素进入、卡片展开 |
| Gentle | cubic-bezier(0.25, 0.1, 0.25, 1) | 桃桃呼吸、等待、照片淡入 |
| Emphasis | cubic-bezier(0.16, 1, 0.3, 1) | 行动卡生长、成功落位 |
| Linear | linear | 极轻进度、循环 shimmer 禁用场景除外 |

禁止：

- 弹簧过冲超过 4px。
- 高频抖动。
- 彩纸爆炸。
- 大面积闪光。
- 旋转超过 8deg 的装饰动效。

## 2. 桃桃生命动态

### 2.1 Idle 呼吸

用途：

- 首页主图边缘。
- 小窝默认状态。
- 聊天空状态。
- 邀请等待。

规格：

- 呼吸 scale：1 -> 1.012 -> 1。
- 垂直位移：0 -> -2px -> 0。
- 周期：2800-4200ms。
- opacity 不变化，避免闪烁。
- 随机眨眼：每 5-9s 触发一次，120-180ms。

实现建议：

- 优先使用 Lottie/Rive/视频透明序列。
- 若只有静态 PNG，使用 transform 动效模拟，但幅度必须极小。
- Reduced Motion 时只保留偶发眨眼或静态姿态。

### 2.2 Attention 注意力

触发：

- 用户进入页面。
- 用户上传照片。
- 用户打开聊天输入。
- 对方进入三方群聊。

动作：

1. 桃桃从 idle 轻停 120ms。
2. 眼神或身体朝向目标移动 2-4px。
3. 出现一句短语或小标记。

禁止：

- 桃桃跳到屏幕中央。
- 挡住照片主体。
- 持续摇晃引导点击。

### 2.3 Thinking 思考

触发：

- 生成晚饭选择。
- 改写一句话。
- 从聊天抽取行动卡。
- 照片对象识别。

动作：

- 桃桃轻微低头或眨眼。
- 旁边出现 3 个微点，点间隔 240ms。
- 最长 6s 后必须给出阶段文案或失败恢复。

文案：

- `桃桃看一下`
- `桃桃想一想`
- `桃桃在找一个温柔说法`

禁止：

- `AI 分析中`
- 百分比进度伪装精确。
- 长时间只显示 spinner。

### 2.4 Positive 正反馈

触发：

- 回忆保存。
- 双方确认。
- 投喂成功。
- 小窝变化。

动作：

- 轻微上浮 4-8px。
- scale 1 -> 1.04 -> 1。
- 小物件或标记淡入。
- 总时长 600-900ms。

规则：

- 开心动态只在正向场景出现。
- 不能在争执降温、安全拦截、支付失败时使用开心动态。

### 2.5 Calm 降温

触发：

- 好好说风险输入。
- 聊天出现高压表达。
- 生成失败。
- 等待对方确认太久。

动作：

- 桃桃从动态回到静止。
- 背景轻降饱和 8%-12%。
- 文案慢 120ms 出现。
- 提供低压力动作：`先不发了`、`换个说法`。

禁止：

- 红色警报。
- 严厉弹窗。
- 桃桃哭泣或责备。

## 3. AI 唤醒动态

唤醒流程是 V2 最重要的 AI 体验，但必须温柔、可信、可退出。

### 3.1 照片进入

步骤：

1. 用户选择照片。
2. 照片从底部或相册位置淡入，时长 280-360ms。
3. 系统等待 200ms，避免立刻“扫描”造成机器感。
4. 出现一个候选对象细线框。

识别标记：

- 线框 opacity 0 -> 1，240ms。
- 线框位置从主体外扩 4px 收到目标边界。
- 标签延迟 120ms 出现。
- 不使用扫描线。

### 3.2 对象确认

动作：

- 用户点击候选对象后，线框变为暖黑或 AI 微标记色。
- 背景照片轻微缩放 1.0 -> 1.015，强调确认。
- 主按钮从 `确认对象` 进入 loading。

失败：

- 识别失败不报技术错误。
- 展示 `桃桃没看清，可以换一张或自己点一下`。
- 支持手动框选或使用默认桃桃。

### 3.3 生成中

进度不使用百分比。使用三段自然语言：

1. `看见这个小东西`
2. `记住你们喜欢它的原因`
3. `让桃桃动起来`

动态：

- 每段 900-1400ms。
- 当前段文字暖黑，已完成段文字中灰 + check。
- 桃桃剪影从模糊到清晰，但不出现花哨能量光。
- 用户可取消，取消后回到照片确认。

### 3.4 预览

动作顺序：

1. 桃桃以 70% opacity 淡入。
2. 眨眼。
3. 轻挥手或靠近照片主体。
4. 出现主按钮 `就用这个桃桃`。

规则：

- 预览只展示 1 个主动态，不自动连播多个动作。
- 用户可点 `再试一次`，但不能无限刷生成造成老虎机感；连续 3 次后提示换图或用默认。

## 4. 三方群聊行动卡生长

行动卡是 AI-native 的核心证据：桃桃把聊天变成下一步。

### 4.1 触发条件

行动卡只能由上下文触发：

- 两人讨论吃什么但未决。
- 一方输入需要改写的话。
- 聊天里出现可记录的小事。
- 对话提到约会时间、地点或计划。

禁止：

- 聊天页常驻功能宫格。
- 每条消息都生成卡。
- 在冲突高风险时生成投喂或娱乐行动。

### 4.2 生长过程

视觉步骤：

1. 桃桃 typing 800-1400ms。
2. 桃桃消息出现：一句识别到的上下文。
3. 消息下方出现 1px 细线或小点，连接到卡片位置。
4. 卡片从 0.96 scale + 0 opacity 生长到位，360-520ms。
5. 卡片内容按顺序出现：标题 -> 证据 -> 选项 -> 主动作。

卡片结构：

- 来源短句：`桃桃听到你们在纠结晚饭。`
- 证据引用：最多 1 行聊天内容。
- 输出：2-3 个选项或 1 个建议。
- 主按钮：`一起选一个`、`换个温柔说法`、`记成回忆`。

### 4.3 状态转换

| 状态 | 动效 |
| --- | --- |
| generating | 卡片骨架只显示标题占位和 2 条短线 |
| ready | 内容淡入，主按钮延迟 80ms |
| selected | 被选项轻压下，check 出现 |
| completed | 卡片高度收敛，显示 `桃桃记住了` |
| dismissed | opacity 0，height collapse 240ms |
| failed | 保留来源消息，卡片显示重试文字动作 |

规则：

- 卡片高度变化必须动画，不突然跳动聊天列表。
- 列表滚动位置要稳定，不能把用户正在读的消息顶走。
- 行动卡最多同时存在 1 张 active；历史卡可以收敛为引用。

## 5. 小窝变化动态

小窝变化要让用户看见“正向行动留下了痕迹”，不是商城奖励。

### 5.1 变化类型

| 类型 | 示例 | 动效 |
| --- | --- | --- |
| Light | 暖灯亮一点 | 600ms 亮度 + 色温变化 |
| Object | 小布丁、小礼物、小床 | 从桃桃旁边或底部轻落位 |
| Memory | 照片摆件 | 照片缩略图淡入到架子上 |
| Note | 感谢小纸条 | 从桃桃身边展开 |
| Mood | 桃桃开心一下 | 600-900ms 正反馈 |

### 5.2 投喂成功

顺序：

1. 支付成功状态确认，按钮变成 `看看小窝变化`。
2. 回到小窝，桃桃进入 positive 动作。
3. 新物件 360-520ms 落位。
4. 小纸条出现：`谢谢你们，桃桃的小窝暖了一点。`
5. 状态收敛，不继续弹付费入口。

禁止：

- 金币飞入。
- 充值成功大弹窗。
- 连续推荐更高档投喂。
- 未投喂导致小窝变暗或桃桃失落。

### 5.3 非付费变化

V2 小窝必须能因行动和回忆变化，不只靠投喂。

触发：

- 保存共同回忆。
- 双方完成晚饭决定。
- 约会计划确认。
- 对方接受邀请。

动效：

- 回忆照片出现在小窝。
- 小灯亮起。
- 桃桃看向新增物件。
- toast：`桃桃把这件事放进小窝了`。

## 6. 邀请等待动态

邀请等待要安静，不制造焦虑。

### 6.1 等待页

视觉：

- 一张邀请来源照片或小窝静态图。
- 桃桃 idle 呼吸。
- 状态文字：`邀请已发出，等对方进来就能一起唤醒桃桃。`
- 主状态按钮：`等待对方确认`。
- 次动作：`复制邀请`、`重新发送`。

动态：

- 桃桃每 6-10s 看向入口一次。
- 状态点每 1200ms 轻淡入淡出。
- 不使用倒计时，不催促用户。

### 6.2 对方进入

动作：

1. 等待状态点停止。
2. 桃桃 attention 动作。
3. 双方头像轻靠近 4-8px。
4. 主按钮变为 `一起继续`。

失败/过期：

- 邀请过期只显示 `这条邀请失效了，再发一次就好`。
- 不做红色警告页。

## 7. 页面转场

### 7.1 默认转场

- 同层页面：横向 slide 16-24px + fade，220-280ms。
- Sheet：底部上滑 280-360ms。
- 照片详情：shared element，照片从卡片扩展到详情。
- 聊天到行动页：行动卡扩展为页面标题区。

### 7.2 返回

- 返回必须保持用户上下文。
- 生成中返回需要确认是否取消。
- 聊天滚动位置保留。
- 小窝变化播放后返回不重复播放，除非用户点 `重看变化`。

## 8. Loading 与 Skeleton

### 8.1 照片加载

- 使用固定比例容器，避免布局跳动。
- 背景 `#F2EEE9`。
- 不使用强 shimmer；可用轻微 opacity 呼吸。

### 8.2 桃桃加载

- 桃桃资产未加载时显示剪影或静态占位。
- 不能出现空白闪一下。
- 加载失败展示安静占位 + `桃桃这会儿没加载出来` + `重试`。

### 8.3 聊天生成

- 桃桃 typing 最长 6s。
- 超过 6s 显示阶段文案。
- 超过 15s 给出失败恢复，不让用户无限等待。

## 9. Reduced Motion

系统减少动态时：

- 呼吸、上浮、落位、卡片生长改为淡入和状态文字。
- 不播放循环动画。
- 识别线框直接出现。
- 小窝变化用静态前后对比和 toast。
- 按钮 pressed 用颜色变化，不 scale。

必须保留：

- loading 状态。
- 成功/失败文字。
- 待确认状态。
- 权限和错误恢复路径。

## 10. 实现建议

### 10.1 技术分层

| 动效类型 | 推荐实现 |
| --- | --- |
| 按钮 pressed、sheet、toast | 原生动画 / CSS transition |
| 桃桃 idle、眨眼、positive | Rive 或 Lottie |
| 复杂唤醒预览 | 短视频透明序列或 Rive |
| 行动卡生长 | 布局动画，需保持聊天滚动稳定 |
| 小窝物件落位 | 原生 transform + opacity |

### 10.2 事件命名

动效应由业务状态驱动：

- `taotao_idle`
- `taotao_attention`
- `taotao_thinking`
- `taotao_positive`
- `taotao_calm`
- `awakening_object_detected`
- `awakening_generation_step_changed`
- `chat_action_card_growing`
- `nest_object_added`
- `invite_partner_joined`

不要用页面临时变量控制核心动效。

## 11. 动效验收

每个动效提交前检查：

- 是否解释了状态变化，而不是装饰？
- 是否保持真实照片主体可见？
- 是否没有扫描线、数据流、霓虹光？
- 是否在 60fps 或目标平台可接受帧率运行？
- 是否不会造成布局跳动？
- 是否有 reduced motion 替代？
- 是否有失败、超时、取消状态？
- 是否符合场景情绪：正向才开心，冲突时降温，等待时安静？
