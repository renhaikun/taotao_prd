# 桃桃 V2 按钮与交互系统

## 0. 设计目标

V2 的按钮不是营销 CTA。按钮只承担当前任务的下一步：确认、继续、发送、保存、邀请、稍后。

交互气质：

- 像一线消费级 App：少按钮、状态清楚、触感稳定。
- 不像增长页：不喊口号、不做转化压迫、不用彩色大 CTA。
- 不像工具后台：不堆边框按钮、不把所有功能同权展示。
- 不像儿童产品：不弹跳、不糖果色、不用大图标按钮阵列。

## 1. 按钮层级

### 1.1 Primary 主按钮

用途：

- 当前页面唯一最重要动作。
- 底部确认动作。
- 支付前的明确投喂确认。
- 邀请另一半、保存回忆、确认唤醒对象、发送给对方。

视觉：

| 属性 | 规格 |
| --- | --- |
| 高度 | 52-56px |
| 最小宽度 | 120px；底部主按钮通常满宽 |
| 圆角 | 16-18px |
| 背景 | 暖黑 `#171411` |
| 文字 | 白色 `#FFFFFF`，16-17px，500-600 weight |
| 图标 | 默认不用；仅发送、保存、邀请可用 18px 线性图标 |
| 阴影 | 无；必要时只用极轻接触阴影 |

文案规则：

- 使用任务动词：`继续`、`确认对象`、`保存回忆`、`发送给对方`、`邀请另一半`。
- 不使用营销文案：`立即开启幸福旅程`、`马上解锁`、`限时体验`。
- 不用超过 8 个中文字；超过则拆成主文案 + 说明。
- 投喂主按钮必须明确金额和对象：`投喂小布丁 ¥1`，不能写 `立即支持桃桃成长`。

状态：

| 状态 | 视觉 | 行为 |
| --- | --- | --- |
| default | 暖黑背景 | 点击执行当前动作 |
| pressed | 背景 `#2A2520`，整体 scale 0.98，80-120ms | 不改变布局 |
| loading | 保持原宽高，文字替换为短状态或 spinner | 防重复点击 |
| disabled | 背景 `#D8D2CC`，文字 `#FFFFFF` | 不可点击，必须有原因 |
| success | 200-400ms 成功反馈后进入下一状态 | 不长时间停留 |
| destructive | 不使用暖黑，改用文字二次确认 | 删除/解绑不能伪装主按钮 |

### 1.2 Secondary 次按钮

V2 次按钮默认是纯文字，不做描边胶囊。

用途：

- 跳过、稍后、重新选择、换一张、用默认桃桃、不给也没关系。
- 与主动作并列但低权重的操作。

视觉：

| 属性 | 规格 |
| --- | --- |
| 高度 | 40-48px 可点击区域 |
| 背景 | 透明 |
| 文字 | `#25221F` 或 `#77716B` |
| 字号 | 15-16px |
| 下划线 | 默认不用；法律/隐私链接可用 |
| 图标 | 默认不用 |

规则：

- 次按钮与主按钮同屏最多 2 个。
- 次按钮不能比主按钮更醒目。
- 不用边框按钮堆叠出“多 CTA”感。
- `不给也没关系` 必须是可点击或明确可退出动作，不只是说明文字。

### 1.3 Tertiary 局部文字动作

用途：

- 图片上的 `换图`、`删除`。
- 聊天消息的 `重发`、`复制`。
- 回忆卡的 `编辑`、`隐藏`。
- 权限说明里的 `去设置`。

视觉：

- 13-15px 文本。
- 暖黑或语义色。
- 点击区域最小 44x44px。
- 不放入胶囊按钮，除非在照片上需要可读底色。

### 1.4 Icon 局部动作按钮

用途：

- 返回、关闭、更多、发送、拍照、语音、图片、删除。

视觉：

| 属性 | 规格 |
| --- | --- |
| 点击区域 | 44x44px |
| 图标 | 20-24px 线性图标 |
| 背景 | 默认透明；照片上用 `rgba(255,255,255,0.72)` |
| 圆角 | 50% 或 12px |
| 颜色 | 暖黑或白色，取决于背景 |

规则：

- 图标必须有无障碍 label。
- 危险动作使用二次确认，不只靠红色图标。
- 不用手绘装饰图标替代标准系统动作。

## 2. 状态按钮

### 2.1 Waiting 待确认按钮

场景：

- 邀请另一半后。
- 回忆等待双方确认。
- 唤醒桃桃等待对方确认。

视觉：

- 文案：`等待对方确认`、`已邀请，等一下`。
- 背景：`#F2EEE9`。
- 文字：`#9A7A3D`。
- 不展示倒计时压迫。
- 可提供纯文字动作：`复制邀请`、`重新发送`。

行为：

- 主按钮变成只读状态，不重复提交。
- 允许用户离开页面。
- 如果状态自动更新，使用轻 toast，不弹窗。

### 2.2 Selected 已选择按钮

场景：

- 吃什么投票。
- 性格选择。
- 照片对象确认。

视觉：

- 背景：`#171411` 或 `#F2EEE9`，依据是否为最终确认。
- 选中标记使用小 check，不用彩色外发光。
- 选项组最多一屏 3-5 个，不做大面积 chip 云。

### 2.3 Completed 完成按钮

场景：

- 回忆已保存。
- 动作已完成。
- 小窝已更新。

视觉：

- 绿色只作为小标记，不做绿色大按钮。
- 文案：`已保存`、`已记住`、`已放进小窝`。
- 600-1200ms 后收敛为普通状态或跳转结果页。

### 2.4 Risk / Blocked 风险按钮

场景：

- 高风险聊天输入。
- 未授权敏感数据。
- 删除、解绑。

规则：

- 风险场景不提供醒目的“继续生成”。
- 删除/解绑使用文字危险动作 + 二次确认 sheet。
- 安全拦截按钮文案用暂停和离开：`先不发了`、`换个说法`、`保存草稿`。

## 3. 交互模式

### 3.1 底部动作区

底部动作区是 V2 最重要的交互容器。

规格：

- 高度随内容：主按钮 52-56px + 顶部 12-16px + 安全区。
- 背景：页面底色或轻微白色遮罩，不用毛玻璃。
- 同时最多一个主按钮 + 一个文字次动作。
- 键盘出现时，底部区跟随键盘上移，不遮挡输入。

页面映射：

| 页面 | 主按钮 | 次动作 |
| --- | --- | --- |
| 唤醒上传 | `选择照片` | `使用默认桃桃` |
| 对象确认 | `确认这个小东西` | `换一张` |
| 生成预览 | `就用这个桃桃` | `再试一次` |
| 聊天发送候选 | `发送给对方` | `再温柔一点` |
| 记小事 | `保存回忆` | `稍后再记` |
| 投喂 | `投喂小布丁 ¥1` | `不给也没关系` |
| 邀请 | `发送邀请` | `复制链接` |

### 3.2 行动卡不是按钮宫格

V2 行动卡只在有上下文时出现。

规则：

- 首页不使用 5 个同权行动卡。
- 聊天中行动卡由消息生成，默认从 0 到 1 张。
- 一张行动卡内最多 1 个主按钮 + 1 个文字次动作。
- 行动卡标题来自用户上下文，不使用模板标题堆叠。
- 今日现场的行动卡默认只露出 1 个主按钮；“换一件事”只能是顶部轻控件，不能形成第二主按钮。
- 今日现场不展示信号列表、功能入口列表或能力宫格。

行动卡结构：

1. 来源：`桃桃从刚才的话里听到...`
2. 主体：晚饭、表达、约会、回忆之一。
3. 证据：一句聊天引用或照片识别点。
4. 动作：`一起选`、`换个说法`、`记下来`。

### 3.3 Sheet 交互

使用场景：

- 投喂选择。
- 删除确认。
- 权限解释。
- 图片来源选择。
- 回忆详情轻编辑。

规格：

- 圆角 24px 以内，只在顶部两个角。
- 顶部不放夸张拖拽条，使用 36x4px 弱指示即可。
- 标题 18-20px。
- 主按钮固定底部，内容可滚动。
- 关闭按钮 44x44px。

禁止：

- 用弹窗展示营销口号。
- 弹窗内放 3 个以上主级按钮。
- 投喂 sheet 做成商品货架。

### 3.4 Toast 与轻反馈

Toast 只反馈结果，不做解释长文。

允许：

- `桃桃记住了`
- `已复制`
- `邀请已发送`
- `图片上传失败，换一张试试`

规格：

- 位置：底部动作区上方或顶部安全区下方。
- 持续：1600-2400ms。
- 不遮挡主按钮。
- 错误 toast 必须提供可恢复路径，不能只报错。

## 4. 页面级交互规范

### 4.1 首页

- 首屏只有一个主动作。
- 主动作来自上下文：晚饭时间优先 `一起决定晚饭`，对方待确认优先 `看看邀请`，新回忆优先 `确认这件小事`。
- 其他入口使用底部导航、文字动作或聊天中的行动卡承接。
- 不做 `立即开始` 这类泛 CTA。

### 4.2 三方群聊

- 输入框右侧发送按钮使用图标按钮。
- 桃桃正在想时，发送按钮仍可用，但避免重复生成同一行动卡。
- 行动卡按钮文案必须是下一步任务：`一起选一个`、`生成 3 个说法`、`记成回忆`。
- 消息失败提供 `重发` 文字动作，不用大红按钮。

### 4.3 唤醒桃桃

- 上传照片后，主按钮从 `选择照片` 变为 `确认对象`。
- AI 识别中按钮进入 loading，但允许 `取消` 文本动作。
- 生成失败时主按钮为 `再试一次`，次动作为 `使用默认桃桃`。
- 双方确认前，按钮状态必须清楚区分 `我已确认` 和 `等待对方确认`。

### 4.4 小窝与投喂

- 小窝页默认不展示金额按钮，用户点击投喂入口后进入 sheet。
- 投喂 sheet 内金额选项是列表，不是商品宫格。
- 支付成功后主按钮不再出现 `继续投喂`，而是 `看看小窝变化`。
- 支付取消后回到小窝，并显示 `不给也没关系` 同等表达。

### 4.5 共同回忆

- 保存回忆是主按钮。
- 删除、隐藏、取消共同可见必须是低权重危险文本动作，并二次确认。
- 待对方确认状态不是按钮诱导，只是状态提示 + `提醒一下` 文本动作。

## 5. Motion 与触感

按钮动效：

- pressed：80-120ms，scale 0.98，opacity 0.92。
- loading：spinner 或 3 点，最短 400ms，避免闪烁。
- success：check 或文字切换 600-1200ms。
- disabled：无动效。

触感：

- 主按钮成功：轻 haptic。
- 删除/失败：不使用强震动。
- 投喂成功：轻 haptic + 小窝变化动效，而不是按钮震动。

Reduced Motion：

- 系统开启减少动态时，scale 动效改为颜色变化。
- loading 保留静态进度文案。
- 成功反馈保留文字和图标，不依赖动画。

## 6. 可访问性

- 所有按钮点击区域不小于 44x44px。
- 主按钮文字对比度必须通过 WCAG AA。
- 图标按钮必须有 label。
- loading 按钮必须向读屏说明 `处理中`。
- disabled 按钮旁边或上方必须说明原因。
- 错误状态不能只靠颜色表达。

## 7. 禁用项

- 大渐变 CTA。
- 描边按钮矩阵。
- 首页功能宫格。
- 两个以上同权主按钮。
- `立即解锁`、`限时开启`、`升级体验`。
- 投喂金额按钮做成充值档位。
- 被拒绝后反复弹出主按钮。
- 负面情绪场景出现投喂 CTA。
- 按钮 hover/pressed 导致布局跳动。

## 8. 开发验收

每个按钮组件必须提供：

- `default`
- `pressed`
- `loading`
- `disabled`
- `success`
- `error` 或失败恢复路径
- `reducedMotion`
- `accessibilityLabel`

每个页面提交前检查：

- 是否只有一个主按钮？
- 主按钮是否对应当前任务，而不是营销转化？
- 次动作是否为纯文字或低权重？
- loading 是否防重复提交？
- disabled 是否说明原因？
- 删除/解绑/支付是否有二次确认？
- 移动端底部安全区是否正确？
