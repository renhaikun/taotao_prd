# 桃桃 V2 视觉方向

## 0. 版本定位

V2 不再延续 V1 的传统情侣 App、萌宠养成、功能入口集合心智。V2 的目标是：

- AI-native：AI 始终参与识别、记录、建议和等待，但不把模型、Agent、分析流程暴露给用户。
- INS 极简：真实照片是主要视觉证据，界面只做留白、排版、动作和轻标记。
- 一线科技 App 质感：控件克制、反馈明确、状态完整，像成熟消费级 App，而不是营销页或儿童互动玩具。
- 动态优先：桃桃的存在感主要来自轻动态、等待、呼吸、眼神、行动卡生长和小窝变化，不靠卡片堆叠和装饰。

一句视觉 thesis：

> 两个人的真实生活照片是主角，桃桃像一个安静但聪明的小生命在旁边识别、记住、推进下一步。

## 1. 核心视觉原则

### 1.1 真实照片主导

照片承担关系真实性，不做装饰背景。

必须做到：

- 首页、共同回忆、唤醒、小窝变化优先使用真实照片或照片级生成图。
- 每个主画面只允许一个照片焦点：一张桌面、一件纪念物、一张旅行照片、一个小窝角落。
- 照片必须保留可识别主体，不做大面积暗化、强模糊、过度裁切或网红滤镜。
- 照片上只允许叠加极少 UI：时间、桃桃识别标记、双方确认状态、轻量动作入口。
- 图片比例固定：首屏主图 4:5 或 3:4，回忆卡 4:5，横向场景 16:10，聊天引用缩略图 1:1。
- 相同页面最多出现一种主照片比例，避免拼贴感。

禁止：

- 用 3D 卡通插图替代真实生活证据。
- 把首页做成照片瀑布流。
- 把照片裁成一堆小卡片组成宫格。
- 用黑色遮罩、强景深、胶片噪点营造高级感。
- 使用情侣模板照、库存甜蜜摆拍、过度商业摄影感图片。

### 1.2 大留白与单焦点

V2 的高级感来自克制的信息密度。

布局规则：

- 每屏只回答一个问题：现在看到什么、桃桃发现了什么、下一步做什么。
- 首屏优先级：真实照片或小窝现场 -> 桃桃轻动态 -> 一个主动作 -> 一个次动作。
- 模块之间使用 24-40px 留白，不用边框和阴影硬分割。
- 文字每组不超过 3 行；超过 3 行必须拆成详情页、sheet 或折叠说明。
- 行动入口不做 2x3 功能宫格。普通页面最多展示 1 个主行动 + 1 个文字次动作；今日现场默认只露出 1 个主行动和 1 个轻切换控件。
- 页面底部固定区只承载当前任务，不放增长文案、活动入口和多按钮堆叠。

视觉层级：

1. 照片主体或桃桃当前动态。
2. 一句桃桃观察或当前状态。
3. 当前最合理的下一步。
4. 可跳过、稍后、重新选择等文本动作。

### 1.3 Apple-like restraint

这里的 Apple-like 指成熟、克制、可用，不是玻璃拟态或系统仿皮。

必须做到：

- 背景以接近白、浅灰、照片本身为主，不靠渐变制造层次。
- 文字颜色使用暖黑、深灰、中灰三级，不用高饱和蓝粉做大面积文字。
- 组件形态简单：无边框页面、轻分隔线、底部 sheet、原生开关、分段控件、文本按钮。
- 控件尺寸稳定，点击后不改变布局尺寸。
- 状态反馈要轻但明确：pressed、loading、disabled、success、error 都必须可见。
- 每个图标都有明确功能，不把图标当装饰。

禁用 UI 材质：

- 玻璃卡片。
- 霓虹渐变。
- 彩色发光描边。
- 大面积毛玻璃。
- 高投影漂浮卡片。
- 过度圆润的玩具块。
- 装饰性光斑、漂浮球、bokeh 背景。

### 1.4 微 AI 识别证据

AI 要可感知，但只能以证据形式轻轻出现。

允许出现：

- 照片里被识别对象旁边的 1px 细线框或短横标记。
- `桃桃看到了这只杯子`、`桃桃记住了这家店` 这类自然语言。
- 生成中出现 2-3 个短步骤：`看照片`、`找共同点`、`变成桃桃的记忆`。
- 聊天里行动卡的生长过程：从一句话抽出标题、时间、选项、确认状态。
- 共同回忆卡上的来源标签：`来自晚饭决定`、`来自聊天`、`来自你们的照片`。

禁止出现：

- AI 扫描线。
- 数据流、矩阵、代码雨。
- 模型名称、Agent 名称、置信度、embedding、路由。
- `AI 正在分析你们的关系`。
- 关系评分、亲密值、风险指数。
- 复杂识别结果面板。

## 2. 色彩系统

### 2.1 基础色

| 用途 | 色值 | 使用规则 |
| --- | --- | --- |
| 页面背景 | `#FAF9F6` | 主背景，带一点暖度，避免冷白刺眼 |
| 内容底 | `#FFFFFF` | sheet、输入区、照片下方信息面 |
| 暖黑 | `#171411` | 主按钮、主标题、关键文本 |
| 主文本 | `#25221F` | 正文和重要说明 |
| 次文本 | `#77716B` | 辅助说明、时间、来源 |
| 弱文本 | `#A9A29B` | placeholder、不可用说明 |
| 细分隔 | `#E8E2DC` | 1px 分隔线，低对比 |
| 轻底色 | `#F2EEE9` | 按下态、局部背景 |

### 2.2 语义色

| 状态 | 色值 | 使用规则 |
| --- | --- | --- |
| 成功 | `#2F7D5C` | 保存、确认、完成，不做大面积背景 |
| 等待 | `#9A7A3D` | 对方待确认、邀请等待 |
| 错误 | `#B24A3A` | 删除、失败、风险提醒 |
| AI 微标记 | `#6E7F8D` | 识别细线、来源标签、低调智能证据 |
| 正向投喂 | `#B56A3E` | 小窝温度、轻感谢，不做购买刺激色 |

### 2.3 色彩禁用项

- 不使用蓝粉渐变作为品牌主视觉。
- 不用紫蓝科技渐变。
- 不把粉色作为情侣感主色。
- 不用金币黄、充值红、限时橙制造付费焦虑。
- 不用多色图标铺满首页。

## 3. 字体与排版

### 3.1 字体

- iOS：系统 `PingFang SC`。
- Android：系统中文无衬线。
- Web/小程序：跟随系统字体。
- 不引入花体、手写体、圆体作为正文。

### 3.2 字号

| 层级 | 字号 | 行高 | 用途 |
| --- | --- | --- | --- |
| 页面标题 | 28-34 | 1.15 | 唤醒、回忆详情等单焦点页面 |
| 区域标题 | 18-22 | 1.25 | 当前对象、今日动作、小窝状态 |
| 正文 | 15-17 | 1.45 | 桃桃一句话、说明、聊天内容 |
| 辅助 | 12-14 | 1.35 | 来源、时间、权限说明 |
| 按钮 | 16-17 | 1.2 | 主按钮、底部动作 |

规则：

- 不使用负字距。
- 不随视口宽度缩放字体。
- 页面标题不进入卡片内部。
- 按钮文字超过 8 个中文字时改写，不压缩字号。

## 4. 图像处理规则

### 4.1 照片风格

照片应像用户真实相册：

- 自然光、低饱和、轻颗粒可接受。
- 主体清楚，背景不过度虚化。
- 人物照片要尊重隐私，默认使用局部生活物件、宠物、食物、场景、纪念物。
- 情绪靠物体和场景表达，不靠夸张贴纸。

### 4.2 裁切规则

- 主体至少占画面 35%-70%。
- 主体不能被按钮、底部栏、识别标记遮挡。
- 头部、宠物脸、纪念物关键部分需要 24px 以上安全边距。
- 所有裁切图需要保留原图引用和裁切参数，便于重新导出。

### 4.3 AI 识别标记

标记形态：

- 1px 线框，圆角 10-14px。
- 小圆点或短横引导线，不做大面积蒙层。
- 文案标签最多 8 个中文字。
- 标签不超过画面宽度 40%。

标记出现时机：

- 唤醒对象确认。
- 桃桃识别到可记录物件。
- 小窝新增物件落位。
- 聊天行动卡生成时引用照片证据。

## 5. 页面方向

### 5.1 首页

首页不是功能首页，而是今天的关系现场。

首屏结构：

1. 顶部关系轻条：双方头像、在一起天数、通知，不抢主视觉。
2. 一张今日照片或小窝现场：真实图像占首屏 55%-68%。
3. 桃桃轻动态：浮在照片边缘或底部，不挡主体。
4. 桃桃一句话：不超过 22 个中文字。
5. 一个主动作：如 `一起决定晚饭`。
6. 两个纯文字次动作：`记一件小事`、`看看小窝`。

禁止：

- 五宫格行动入口。
- 首页大口号。
- 会员、投喂、活动先于今日行动。
- 一屏塞入回忆、聊天、小窝、设置等所有入口。

### 5.2 三方群聊

聊天必须像两个人和桃桃同处一个群，而不是 AI 助手页。

视觉规则：

- 她/他消息用极浅底或无底文本块，靠头像、缩进、时间区分。
- 桃桃消息更像旁白和行动建议，不做机器人气泡。
- 行动卡从聊天中生长，不作为预置功能卡片常驻。
- 输入区保持原生、干净，快捷动作最多 3 个并可横滑。

### 5.3 唤醒桃桃

唤醒是从照片里建立共同来源，不是头像生成器。

视觉规则：

- 上传后立即进入照片大图确认。
- AI 识别标记只圈出一个候选对象。
- 生成进度用平静步骤，不用炫技动画。
- 预览时先看桃桃动起来，再看名字和口吻设置。

### 5.4 小窝

小窝是关系现场的安静角落，不是商城。

视觉规则：

- 小窝主画面使用照片级空间，桃桃动态是唯一焦点。
- 投喂入口默认收敛，只在正向场景或用户主动进入时出现。
- 小窝变化要可见：灯变暖、一件小物出现、照片被摆上架。
- 不投喂时小窝不变差。

## 6. 禁用清单

### 6.1 视觉禁用

- 花哨渐变。
- 玻璃卡片。
- 功能宫格。
- 大面积卡通角色。
- 儿童玩具感。
- 宠物游戏感。
- 营销落地页式首屏。
- 手绘爱心、贴纸、星星、彩带堆叠。
- 闪烁扫描线、数据流、AI 光效。
- 复杂插画做主视觉。

### 6.2 文案禁用

- `AI 助手`
- `智能助手`
- `关系分析`
- `心灵共振`
- `评理`
- `裁决`
- `亲密值`
- `关系评分`
- `解锁高级能力`
- `限时投喂`
- `不投喂会影响桃桃`

### 6.3 交互禁用

- 首页用多个大 CTA 拉转化。
- 用支付色强调投喂。
- 用弹窗打断聊天核心流程。
- 在吵架、冷战、伤心场景出现投喂。
- 识别结果未经用户确认自动写入共同回忆。
- 未授权读取外部聊天、定位、相册。

## 7. 设计验收

每个 V2 页面提交前必须回答：

- 第一眼是不是看到真实生活证据，而不是功能集合？
- 当前屏是否只有一个视觉焦点？
- 桃桃是否以动态或一句话存在，而不是用大卡通占满页面？
- AI 是否只以微证据出现，没有技术术语？
- 主动作是否是当前任务，不是营销 CTA？
- 次动作是否为纯文字或低权重控件？
- 是否去除了渐变、玻璃、宫格和儿童宠物感？
- 空、加载、错误、权限、待确认状态是否都有同一套克制视觉？
