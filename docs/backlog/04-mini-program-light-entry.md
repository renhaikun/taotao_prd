# 小程序轻端分册

## 目标

小程序只做轻入口，不复制完整 App。它负责邀请确认、绑定提示、轻处理和回 App。

## 能做

| 场景 | 小程序动作 |
| --- | --- |
| 微信分享卡打开 | 授权、登录、确认加入 |
| 手机号绑定提示 | 查看邀请、确认或拒绝 |
| 上书待处理 | 轻查看、准奏、驳回 |
| 需要复杂修改 | 引导打开 App |

## 不做

- 完整三人聊天。
- 桃桃形象生成。
- 小窝完整浏览。
- 深设置。
- 投喂支付闭环。

## 状态

| 状态 | UI | 后续 |
| --- | --- | --- |
| wx_guest | 微信授权 | 授权成功后恢复邀请 |
| invite_review | 邀请确认 | 加入或暂不加入 |
| phone_prompt_review | 手机号绑定确认 | 确认或不是本人 |
| light_petition_review | 上书轻处理 | 准奏、驳回、打开 App 修改 |
| open_app | 回 App | 成功打开或复制链接 |

## 核心规则

- 小程序不能绕过手机号主账号。
- 邀请确认前不建立关系。
- 轻处理后的状态必须同步回 App。
- 小程序里不出现 App 才有的复杂页面。

## 验收

- 分享卡打开后能恢复邀请上下文。
- 手机号提示必须由接收方确认。
- 已绑定用户再次打开邀请不能误绑新关系。
- 回 App 失败时能复制链接或稍后处理。
