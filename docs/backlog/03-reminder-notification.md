# 提醒通知分册

## 目标

提醒只服务小事推进和关系绑定，不把桃桃做成提醒机器人。

## 提醒来源

| 来源 | 是否需要提醒 | 说明 |
| --- | --- | --- |
| 上书待处理 | 是 | 提醒接收方 |
| 小事时间临近 | 是 | 参与人可收到 |
| 对方已看未处理 | 可选 | 不频繁打扰 |
| 发送失败 | 否 | 留在原消息现场 |
| 普通聊天 | 否 | 不推送 |

## 页面和入口

| 位置 | 展示 |
| --- | --- |
| 顶部胶囊 | 当天最重要待处理 |
| 今天的小事抽屉 | 全部待处理和已定 |
| 系统通知 | 离开 App 后的必要提醒 |
| 设置 | 提醒频率和安静时间 |

## 核心规则

- 发送失败不进入通知中心，像微信消息失败一样停在原处。
- 桃桃不在群聊里播报“对方看过了”。
- 未绑定用户只接收邀请相关提醒。
- 安静时间内只保留高优先级提醒。

## 状态

| 状态 | UI | 后续 |
| --- | --- | --- |
| scheduled | 已设置提醒 | 可修改、取消 |
| due_soon | 快到了 | 打开小事详情 |
| missed | 已过时间 | 延后、收起 |
| permission_off | 无通知权限 | 引导打开系统权限 |
| quiet_suppressed | 安静时间内 | 到时间后合并提醒 |

## 验收

- 关闭系统通知后，App 内顶部胶囊仍可找回小事。
- 通知点击必须回到对应小事详情。
- 普通聊天不触发提醒。
- 所有提醒都能找到来源对象。
