# 桃桃页分册

## 目标

桃桃页是从聊天里点开桃桃后的轻档案页，不是一级 Tab，也不是设置页。MVP 承接“看见桃桃”“修改名字”和“日历回看”，不在本阶段开放形象、来源、装扮或生成能力。

当前评审原型入口：

- `http://127.0.0.1:5175/?mode=taotao`
- 版本：V7.9 桃桃页评审状态板
- 数据源：`prototype/src/data/v7TaotaoBoardData.js`
- 组件：`prototype/src/components/TaotaoLifeBoard.jsx`

旧 V6 `route=taotao` 不再作为主线桃桃页入口。桃桃页必须作为 V7.x 分册状态板单独评审，再进入真实开发排期。

## MVP 范围

| 能力 | 本阶段是否做 | 说明 |
| --- | --- | --- |
| 看见桃桃 | 做 | 展示默认桃桃形象和当前名字 |
| 修改名字 | 做 | 默认叫桃桃，用户可在页面内直接改名 |
| 日历回看 | 做 | 用日历承接聊天里自然形成的小事、回看和轻沉淀 |
| 换形象 | 不做 | 后续作为 AI 装扮/生成能力规划 |
| 上传照片生成桃桃 | 不做 | 成本和开发负担高，后置到商业化阶段 |
| 调整说话方式 | 不做 | 先由聊天 agent 能力承接 |
| 装扮/会员 | 不做 | 后续独立规划付费链路 |

## 入口

| 入口 | 进入后 |
| --- | --- |
| 聊天顶部桃桃头像 | 打开桃桃页 |
| 桃桃消息头像 | 打开桃桃页 |
| 设置里的桃桃资料 | 后续设置分册处理，MVP 不做 |

## 页面结构

| 区块 | 用户看到 | 动作 |
| --- | --- | --- |
| 桃桃形象 | 默认桃桃形象 | 查看 |
| 名字 | 默认 `桃桃`，用户可改名 | 页面内直接编辑 |
| 日历 | 7 日轻日期条、月历展开和当天小事 | 点日期或小事回到聊天源位置 |
| 未来能力入口 | 暂不展示或灰度隐藏 | 不进入 MVP |

## 核心规则

- 用户没有改名时，全局默认叫桃桃。
- 用户改名后，用户端显示新名字；文档和代码对象仍可使用 `taotao` 作为内部对象名。
- 上传照片、生成形象、装扮、会员能力全部不进入 MVP。
- 桃桃页不承接聊天主流程，主要动作仍回到聊天。
- 桃桃页不放底部 Tab，也不放小窝 Tab。
- 日历不是任务列表，也不是独立小窝；它负责时间感、索引、回看和小事轻沉淀。
- 不用“暂未开放”“敬请期待”这类占位文案污染用户界面；没做的能力先不展示。

## 后置商业化方向

后续可以把形象和装扮做成独立付费能力：

- 用户上传自己、宠物或纪念物照片。
- AI 生成专属桃桃形象。
- 生成节日装扮、情侣主题装扮或动态头像。
- 会员或单次付费解锁生成额度。

这部分后续单独出分册，不和 MVP 桃桃页混在一起。

## 状态

| 状态 | UI | 后续 |
| --- | --- | --- |
| chat_entry | 聊天关系栏或桃桃头像进入 | 不改变聊天上下文 |
| default_ready | 默认桃桃可用 | 可改名、回聊天 |
| editing_name | 名字行编辑态 | 保存或返回 |
| name_saved | 名字行保存成功 | 回到展示态 |
| name_error | 名字保存失败 | 保留原名，可重试 |
| calendar_open | 日历展开 | 点回聊天原位置或小事详情 |
| calendar_empty | 日历还没有小事 | 回到聊天继续使用 |
| calendar_return_chat | 回到聊天源位置 | 保持消息流和胶囊索引 |

## 验收

- 未上传照片也能完整使用聊天。
- 改名后聊天头像、桃桃页、顶部胶囊里的显示名一致。
- 桃桃页没有换形象、上传照片、生成、装扮、会员入口。
- 桃桃页没有底部 Tab。
- 用户端不出现模型、识别、3D 任务等内部词。
- `?mode=taotao` 根节点必须是 `data-surface="taotao"` 与 `data-prototype-version="v7.9"`。
- 手机稿只展示用户真实看到的内容；API、埋点、状态字段只能出现在状态板说明区。
