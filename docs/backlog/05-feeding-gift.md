# 投喂礼物分册

## 目标

投喂是情绪合适时的轻付费反馈，不是会员墙，也不是强制功能入口。

## 触发时机

| 场景 | 是否可触发 |
| --- | --- |
| 双方刚完成一件开心小事 | 可触发 |
| 桃桃帮忙生成了有用结果 | 可触发 |
| 争执、负面情绪、安全边界 | 禁止 |
| 用户刚失败或被拒绝 | 禁止 |

## 展示方式

桃桃可以自然撒娇，但只在低打扰位置出现。用户可以随意金额投喂，也可以关闭。

## 核心规则

- 不做会员制作为早期主路径。
- 不诱导未成年人或情绪脆弱用户付费。
- 不在争执和安全场景触发。
- 支付失败不改变桃桃状态。
- 投喂后给出轻反馈，不做复杂数值养成。

## 状态

| 状态 | UI | 后续 |
| --- | --- | --- |
| feed_prompt | 轻投喂入口 | 投喂、以后再说 |
| amount_select | 金额选择 | 支付或取消 |
| paying | 支付中 | 成功或失败 |
| paid | 桃桃反馈 | 回聊天 |
| failed | 支付失败 | 重试或取消 |

## 验收

- 负面场景不出现投喂。
- 取消投喂不影响当前功能。
- 支付失败不生成礼物结果。
- 投喂入口不会压住聊天输入。
