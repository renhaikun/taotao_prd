# 产品定位与 MVP 范围

## 一句话定位

桃桃是一款情侣共同养成的拟人化关系小伙伴。它由两个人一起唤醒、一起投喂、一起养大，陪情侣处理日常小事、聊天表达、约会灵感、共同记忆和轻度关系修复。

## 产品不是

- 不是情侣打卡工具。
- 不是纪念日工具。
- 不是情侣项目管理软件。
- 不是吵架评理 App。
- 不是 AI 问答助手。
- 不是金融导流 App。
- 不是会员制订阅工具。

## 产品是

- 两个人共同拥有的拟人化小伙伴。
- 一个能把日常小事变得有温度的关系界面。
- 一个从用户照片中被唤醒、能 3D 动态化呈现的情感角色。
- 一个懂双方偏好、表达习惯、共同小事和关系节奏的陪伴者。
- 一个通过投喂、小窝、装扮、节日礼物形成轻商业化的消费级 AI 产品。

## 核心用户

首发面向中国本土稳定情侣，优先覆盖以下关系阶段：

- 刚建立稳定关系，希望拥有专属双人空间。
- 异地或半异地，需要更轻的日常陪伴。
- 同城情侣，需要一起决定吃什么、去哪玩、怎么表达。
- 准备同居、订婚或新婚，希望建立共同生活节奏。

首发不优先覆盖：

- 高冲突关系。
- 分手挽回。
- 婚恋咨询。
- 心理治疗。
- 家庭财务管理。

## 首发 MVP 目标

MVP 不追求功能多，而追求一个强闭环：

`唤醒桃桃 -> 和桃桃一起做日常小事 -> 形成正向互动 -> 沉淀共同记忆 -> 投喂和养成桃桃 -> 下次继续回来`

MVP 要证明三件事：

1. 情侣愿意共同绑定并一起创建桃桃。
2. 情侣愿意在吃饭、聊天、约会、记录小事等场景里找桃桃。
3. 情侣愿意在高情绪价值时投喂桃桃，形成非会员制轻付费。

## 首发功能范围

必须包含：

- 情侣绑定。
- 上传照片唤醒桃桃。
- 桃桃 3D 动态形象预览。
- 桃桃名字、性格、说话风格设置。
- 首页行动优先结构。
- 聊天入口。
- 吃什么。
- 帮我好好说。
- 约会灵感。
- 记一件小事。
- 投喂桃桃。
- 共同回忆。
- 小窝与装扮基础能力。
- 基础安全边界与用户授权。

暂不包含：

- 会员订阅。
- 愿望罐。
- 基金或理财入口。
- 定位追踪。
- 情侣打分。
- 谁对谁错裁决。
- 历史争吵排名。
- 复杂心理测评。
- 多个 AI 角色暴露给用户。

## 首发北极星指标

每周至少完成一次 `情侣共同使用桃桃完成日常动作` 的活跃关系数。

这里的日常动作包括：

- 一起决定吃什么。
- 和桃桃聊天。
- 生成或修改一句表达。
- 创建约会灵感。
- 记录一件共同小事。
- 投喂桃桃。

## 关键指标

- 情侣绑定完成率。
- 上传照片唤醒桃桃完成率。
- 另一半确认率。
- 首日聊天触发率。
- 每周共同动作次数。
- 共同回忆创建数。
- 投喂触发率。
- 投喂支付转化率。
- 4 周活跃关系留存。
- 用户主动修改桃桃名字、性格、头像或小窝的比例。

## 产品原则

1. 拟人化优先：用户感知到的是桃桃，不是技术。
2. 行动优先：先让情侣一起做事，再展示回忆。
3. 低压力：不打卡、不惩罚、不排名。
4. 正向付费：只在开心、完成、被帮助后的场景里触发投喂。
5. 安全边界：不监控、不站边、不羞辱、不诱导控制。
