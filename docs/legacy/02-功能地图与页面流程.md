# 功能地图与页面流程

## 信息架构

首发底部导航为五个入口：

1. `首页`
2. `回忆`
3. `桃桃`
4. `小窝`
5. `我们`

底部中间必须是 `桃桃`。它代表她、他和桃桃都在场的三方群聊现场，不是问答工具入口。

未绑定状态下，底部入口可以露出方向感，但不能打开完整能力。聊天、回忆、小窝、唤醒、相册和我们页都要回到邀请路径，只展示邀请另一半和少量轻预告。

## 首页层级

首页必须遵守 `行动优先，回忆后置`。

推荐顺序：

1. 顶部关系信息：双方头像、在一起天数、通知。
2. 桃桃登场：3D 动态桃桃、桃桃开口、桃桃先递出的今日主动作、桃桃的小口袋。
3. 今天我们可以一起：一个主建议，例如今晚先定吃什么，再给好好说、记小事、约会灵感等轻选项。
4. 共同回忆：最近 3-4 个回忆卡片。
5. 轻提示：桃桃的今日小愿望、节日提醒或小窝变化。

禁止顺序：

- 先展示共同回忆，再展示今天可以做什么。
- 首页变成照片流。
- 首页变成纪念日工具。
- 首页变成 AI 能力展示页。
- 未绑定时展示完整聊天、共同回忆、投喂、唤醒完成态或边界管理能力。

## 核心页面

### 首页

目标：让用户第一眼看到桃桃，并立刻知道今天可以一起做什么。

核心模块：

- 桃桃形象。
- 桃桃说话气泡。
- 桃桃先递出的今日主动作。
- 桃桃的小口袋。
- 今天我们可以一起，但不铺成功能宫格。
- 共同回忆。

关键交互：

- 点击桃桃进入聊天。
- 点击 `吃什么` 进入晚饭决策流。
- 点击底部 `桃桃` 进入三方群聊。
- 点击 `约会灵感` 进入计划生成。
- 点击 `记一件小事` 创建回忆。
- 在合适场景点击桃桃的小口袋或小窝入口，进入小零花选择。
- 点击桃桃的小口袋查看桃桃收到过的小开心和小窝反馈。

未绑定时：

- 首页主动作是邀请另一半。
- 今日行动只作为轻预告出现。
- 点击需要双方共同完成的入口时，回到邀请路径。

### 聊天

目标：聊天页是 `情侣双方 + 桃桃` 的三方群聊。桃桃作为一个会说话的小伙伴参与两个人的日常，而不是单人问答工具。

聊天支持：

- 她、他、桃桃三方同时在一个群聊语境里出现。
- 普通聊天。
- 吃什么。
- 帮我好好说。
- 约会灵感。
- 送礼灵感。
- 安抚一下。
- 记录小事。
- 回忆引用。

聊天页顶部用她、他、桃桃三方头像和桃桃状态建立在场感。吃什么、好好说、记小事只作为轻入口；约会灵感、送礼灵感和安抚一下由桃桃在聊天流里递出来，避免把群聊变成功能入口堆叠。

聊天入口文案：

- `桃桃在这里`
- `来和桃桃聊聊`
- `桃桃帮你们想想`
- `我们和桃桃`

避免文案：

- 问答工具感。
- 助手工具感。
- 诊断关系感。
- 神秘信号感。

### 回忆

目标：沉淀情侣与桃桃一起经历过的小事。

回忆类型：

- 日常小事。
- 约会计划。
- 吃饭决定。
- 礼物灵感。
- 甜蜜表达。
- 纪念日。
- 桃桃成长瞬间。

回忆卡片字段：

- 标题。
- 日期。
- 来源场景。
- 图片或贴纸。
- 桃桃的一句话。
- 双方是否共同确认。

### 小窝

目标：让用户看见桃桃真的住在两个人的小窝里，并把共同回忆、小零花和小物件变成一个连续空间。

小窝包含：

- 主舞台：桃桃当前形象、小窝背景、暖光数值和最新共同回忆。
- 今日小窝：共同回忆带来的暖光变化，以及小夜灯、小布丁、礼物盒等小物件。
- 小零花：桃桃撒娇要一点小布丁、小玩具或小礼物，不给也没关系。
- 小窝记录：App 和小程序送给桃桃的小零花，都回到同一条小窝记录里。
- 边界提醒：情绪较重、争执或冷战时，桃桃先陪两个人缓一缓。

小窝不是商城首页。付费入口要温柔、轻量、场景化。

### 我们

目标：展示两个人和桃桃的小窝概览，并把资料、偏好和边界收进二级页。

包含：

- 今天一起做一件小事。
- 在一起天数、共同回忆和小窝暖光。
- 最近留下的事。
- 桃桃收到过的小开心。
- 桃桃的样子：名字、性格、照片与形象、说话方式。
- 我们的小约定：双方资料、纪念日、偏好和重新邀请确认。
- 边界与安心：相册与相机、小事提醒、来源照片可清掉、共同内容需要双方再次确认。

桃桃的样子不是静态说明。名字、性格和说话风格保存后，要同步影响首页、三方群聊、小窝、小程序、旅程、提醒、相册、记小事和回忆详情里的桃桃称呼与描述。

### 小程序

目标：承接微信里的轻量进入和回流，不复制完整 App。

状态规则：

- 未进入：微信授权，让桃桃先认得当前用户。
- 已进入未绑定：展示小窝卡，继续邀请对方。
- 已绑定：展示微信群聊预览、回 App 继续、轻选择发回三方群聊。

小程序不能展示完整 App 功能集合。微信里完成的轻选择回到 App 的三方群聊；投喂和小窝成长只在 App 内完成。

### 共同相册

目标：承接共同回忆照片、唤醒桃桃的来源照片和相册边界。

包含：

- 最新共同回忆。
- 最近照片。
- 可以用来唤醒桃桃的照片。
- 源照片用途说明。
- 相册与相机授权管理。

共同照片带回唤醒流程时，要保留照片来源和对象草稿，不能让用户重新从空白开始。

### 旅程

目标：承接从进入、邀请、唤醒到第一次共同动作的进度，但不做功能目录。

未绑定时只展示进入和邀请指引；已绑定时只推荐一个今天动作和少量备选。

## 核心流程

### 1. 首次进入与绑定

App 入口：

`打开 App -> 手机号 -> 验证码 -> 邀请另一半 -> 对方打开邀请 -> 双方确认 -> 进入同一个小窝`

小程序入口：

`打开小程序 -> 微信授权 -> 认得当前用户 -> 邀请另一半或回到已有小窝 -> 双方确认 -> 进入同一个小窝`

### 2. 唤醒桃桃与全局资料

`同一个小窝 -> 唤醒桃桃 -> 选择照片来源 -> 选择醒来的对象 -> 设置性格、名字和说话方式 -> 选择动作 -> 另一半确认 -> 完成 -> 首页、三方群聊、小窝、我们页、小程序、提醒、相册和回忆同步桃桃资料`

从相册进入唤醒时：

`唤醒桃桃 -> 打开相册 -> 选择共同照片 -> 回到唤醒流程 -> 保留照片来源和对象草稿 -> 继续设置性格和动作`

关键原则：

- App 和小程序是两个入口，不在登录页互相混用。
- 未进入和未绑定分开处理：未进入先完成身份确认，已进入未绑定再进入邀请。
- 邀请绑定发生在登录之后，不把关系创建压进登录流程。
- 允许一方先创建，另一方后确认。
- 邀请页要让用户先看见小窝卡和一个主动作；分享方式只做轻切换，不铺成功能选择面板。
- 如果是对方发来的邀请，登录后要保留待确认状态，不能重置成未绑定。
- 未绑定时不能进入完整桃桃关系体验。
- 绑定前只看邀请和轻预告，不能产生共同记忆、共同投喂、唤醒完成态或完整三方群聊内容。

### 3. 唤醒桃桃细节

`选择照片来源 -> 选择醒来的对象 -> 设置性格、名字和说话方式 -> 预览动作 -> 另一半确认 -> 完成`

照片来源：

- 宠物。
- 纪念物。
- 合照中的物件。
- 玩偶。
- 礼物。
- 其他双方认可的共同对象。

没有照片时：

- 可以选择默认形象。
- 后续可重新上传并二次唤醒。

资料同步：

- 默认名字是桃桃，用户可以改名。
- 改名后，首页、三方群聊、小窝、投喂弹层、我们页、小程序、旅程、提醒、相册、记小事和回忆详情都使用同一个名字。
- 宠物、纪念物、玩偶、礼物和共同照片里的小物件都可以生成动态小伙伴。

### 4. 吃什么

`点击吃什么 -> 选择限制条件 -> 选一个晚饭方向 -> 放回三方群聊确认 -> 可记录为回忆 -> 合适时可触发投喂`

输入条件：

- 时间。
- 预算。
- 口味。
- 是否外卖。
- 是否下厨。
- 禁忌。
- 今日心情。

输出：

- 推荐菜或餐厅方向。
- 原因。
- 可执行下一步。
- 桃桃拟人化评论。

### 5. 聊天

`点击桃桃 -> 进入三方群聊 -> 任一方输入 -> 桃桃识别场景 -> 给出拟人化回应 -> 必要时递出行动卡片`

行动卡片包括：

- 去决定晚饭。
- 换个温柔说法。
- 生成约会灵感。
- 记入回忆。
- 投喂桃桃。

### 6. 帮我好好说

`输入原话 -> 选择语气 -> 桃桃生成 2-3 个版本 -> 用户选择 -> 发送给对方或复制`

语气：

- 温柔一点。
- 撒娇一点。
- 认真一点。
- 不委屈自己。
- 先降温。

边界：

- 不替用户操控对方。
- 不生成威胁、羞辱、冷暴力文案。
- 不判断哪一方正确。

### 7. 约会灵感

`点击约会灵感 -> 选择时间/预算/心情 -> 选一个轻计划 -> 放回三方群聊确认 -> 可记成小愿望或共同回忆`

计划应轻量，不做复杂日程工具。

界面上第一眼应该是一张主建议卡和少量备选，不要让筛选 chips 抢走主建议。

### 8. 记一件小事

`从聊天或行动页进入 -> 带入刚刚那件事 -> 显示回忆明信片 -> 用户可展开调整标题、图片和文字 -> 保存为共同回忆 -> 另一半确认 -> 可带回聊天`

吃什么、约会灵感和送礼灵感进入记小事时，要自动带入刚刚选中的内容，不让用户从空白开始。

`点击记一件小事 -> 输入文字/照片 -> 桃桃补一句话 -> 双方可确认 -> 进入共同回忆`

回忆详情优先展示图片、桃桃一句话和双方确认条，避免做成来源、心情、下一步等数据表格。

桃桃补充的是温度，不是总结报告。

### 9. 投喂桃桃

`正向场景完成 -> 桃桃撒娇提示 -> 用户选择小零花/道具 -> 桃桃确认 -> 小窝记录和暖光变化`

触发场景：

- 晚饭决定完成。
- 聊天帮助完成。
- 好好说文案被采纳。
- 约会计划生成。
- 共同回忆创建。
- 节日或纪念日。
- 小程序里送给桃桃的小零花。

禁止触发场景：

- 吵架中。
- 冷战中。
- 一方明显伤心。
- 高风险关系冲突。

完成后：

- 桃桃反馈要像小伙伴收到礼物，而不是交易结果。
- 小窝暖光和小物件状态同步变化。
- App 内完成的投喂要写回小窝记录，并直达小窝查看变化。
- 用户拒绝后不连续催促。

### 10. 小程序轻体验

`打开小程序 -> 微信授权 -> 未绑定时分享小窝卡 / 已绑定时展示轻群聊 -> 把轻选择发回三方群聊 -> 回 App 继续聊天或查看小窝`

边界：

- 小程序只承接微信里的轻量进入和回流。
- 未绑定只做授权、邀请和轻预告。
- 已绑定只保留微信群聊预览、回 App 继续、把轻选择发回三方群聊。
- 不在小程序里复制完整首页、完整回忆、完整我们页或完整唤醒流程。

### 11. 共同相册与我们页

共同相册：

`共同回忆/来源照片 -> 查看用途 -> 说明边界 -> 选择照片带回唤醒 -> 保留照片来源和对象草稿`

我们页：

`小窝概览 -> 桃桃的样子 -> 我们的小约定 -> 少打扰一点 -> 边界与安心`

原则：

- 我们页是两个人与桃桃的小窝概览，不是功能说明页。
- 桃桃名字、性格、照片与形象、说话方式保存后，全链路同步。
- 解除或重新绑定要回到邀请路径，避免停在已绑定专属页面。
