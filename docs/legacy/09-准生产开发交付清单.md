## 开发优先级

### M0 必须先做

- App 手机号进入：手机号、验证码、轻资料确认。
- App 情侣绑定：发小窝卡、对方看过、双方确认、绑定成功。
- 首页：桃桃主视觉、今天一起做一件事、未绑定轻预告。
- 三方群聊：她、他、桃桃同场；双方身份切换；发送中、失败、重试。
- 吃什么：轻选择、放回三方群聊、可记录为小事。
- 好好说：不触发投喂，只回到三方群聊继续说。
- 记小事和回忆详情：保存、等待确认、双方确认、带回聊天。
- 小窝与投喂：只在正向场景触发；成功后能回群聊或首页。
- 小程序轻入口：微信授权、微信内发小窝卡、对方看过、一起确认、已绑定后回 App 继续聊或同步一个轻选择。

### M1 可后置

- 约会灵感。
- 送礼灵感。
- 安抚一下。
- 共同相册。
- 我们页完整边界。
- 提醒页少打扰与隐私结果态。
- 唤醒桃桃全流程的真实媒体上传和生成队列。

### 先 mock，后接真实能力

- 3D/2.5D 生成与动态图。
- 真实支付。
- 微信/短信/登录服务。
- 相册系统权限。
- AI 编排、记忆存储、安全策略。

## 产品原则

- 桃桃是拟人化小伙伴，不是工具页、裁判或关系诊断器。
- 用户第一眼应看到“今天一起做事”，不是功能列表。
- 小程序只做轻入口，不复制完整 App。
- 投喂只在开心、完成、被帮助后的正向场景出现。
- 情绪较重时先陪伴和降温，不撒娇要钱。
- 所有用户可见图片必须来自资产包或真实上传内容，不用 UI 截图冒充生产资产。

## 开发验收

- `npm run build` 必须通过。
- 首页、注册、邀请、三方群聊、小程序、小窝核心链路必须能连续走通。
- `production-v2/asset-manifest.json` 中列出的运行时资产必须存在。
- QA 截图只截手机画面本体，不混入设计 rail、过程图或多屏拼图。
- 当前默认命名使用“桃桃”；自定义昵称只能出现在同步证明和改名后的真实状态。
