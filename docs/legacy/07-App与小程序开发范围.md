# App 与小程序开发范围

## 平台目标

首发目标为：

- 移动 App。
- 微信小程序。

两端共享产品逻辑和核心后端，但页面深度不同。

## App 与小程序差异

### App

适合承载完整体验：

- 完整唤醒桃桃流程。
- 更强 3D 动态展示。
- 聊天。
- 小窝。
- 回忆。
- 投喂。
- 后续共同目标。

### 小程序

适合承载轻量体验：

- 微信授权进入。
- 微信内分享小窝卡。
- 对方看过和一起确认。
- 已绑定回流。
- 轻量聊天预览。
- 轻量吃什么。
- 轻选择回流。

小程序不复制完整 App。未绑定时只做微信授权、小窝卡分享和邀请确认；已绑定后保留四个轻入口：

- 今天：快速选择今天先做一件事。
- 群聊：看到她、他和桃桃的轻量对话，并能回 App 继续。
- 吃什么：完成轻量晚饭选择，并同步到三方群聊。
- 投喂：开心时投喂桃桃，并回 App 查看小窝变化。

小程序不适合一开始做完整唤醒、深小窝装修、复杂设置和长期资料管理。唤醒可以承接来源照片或回 App 继续，但完整流程放在 App。

## MVP 模块拆分

### 1. 账号与绑定

能力：

- App 手机号与验证码进入。
- 小程序微信授权进入。
- 登录后邀请另一半。
- 生成邀请。
- 另一半确认。
- 回到已有小窝。
- 解除绑定。

### 2. 桃桃生成

能力：

- 上传照片。
- 识别对象。
- 选择性格方向。
- 生成 3D/2.5D 桃桃形象。
- 预览。
- 设置名字和性格。
- 双方确认后完成。
- 保存形象资源。

MVP 可以先使用生成式图片和轻动效，不必一开始做完整实时 3D。

小程序首发不做完整桃桃生成，只保留回流与轻承接。

### 3. 首页

能力：

- 展示双方头像。
- 展示在一起天数。
- 展示桃桃。
- 展示今天我们可以一起。
- 展示桃桃先递出的一个主建议，以及好好说、记小事、约会灵感等轻选项。
- 展示桃桃的小口袋。
- 展示共同回忆。

### 4. 聊天

能力：

- 情侣双方与桃桃组成三方群聊。
- 支持她和他的发言身份。
- 场景识别。
- 快捷行动卡片。
- 对话历史。
- 安全拦截。

### 5. 吃什么

能力：

- 采集偏好。
- 生成建议。
- 双方选择。
- 记录结果。
- 可转回忆。
- 可触发投喂。

### 6. 好好说

能力：

- 输入原话。
- 选择语气。
- 生成多个版本。
- 复制或发送。
- 安全检查。

### 7. 回忆

能力：

- 创建回忆。
- 展示回忆。
- 上传图片。
- 桃桃补一句话。
- 双方确认。

### 8. 送礼灵感与安抚一下

能力：

- 给出轻量小惊喜建议。
- 给出第一句安抚表达。
- 发送回三方群聊。
- 记录为小愿望或共同小事。
- 情绪较重时，桃桃先陪两个人缓一缓，不撒娇要小零花。

### 9. 小窝与投喂

能力：

- 展示小窝。
- 展示小窝成长。
- 展示装扮与礼物。
- 展示桃桃收到过的小开心。
- 投喂档位。
- 支付。
- 投喂反馈。
- 小窝变化。

### 10. 我们与边界

能力：

- 桃桃的样子。
- 双方资料。
- 关系偏好。
- 边界与安心。
- 来源照片可清掉。
- 共同内容需要双方再次确认。
- 解除绑定。

## 后端核心服务

建议拆分：

- User Service。
- Couple Service。
- Taotao Profile Service。
- Media Generation Service。
- Chat Orchestration Service。
- Memory Service。
- Feeding/Nest Service。
- Safety Service。
- Payment Service。

## 首发确认标准

产品确认：

- 用户可以完成情侣绑定。
- App 用户可以在验证码后完成轻资料确认，再发小窝卡。
- 小程序用户可以在微信内完成发小窝卡、对方看过和一起确认。
- 用户可以上传照片并唤醒桃桃。
- 用户可以看到 3D/2.5D 动态桃桃。
- 首页先展示今天可以一起做什么，再展示共同回忆。
- 用户可以和桃桃聊天。
- 用户可以完成吃什么流程。
- 用户可以使用好好说。
- 用户可以使用送礼灵感。
- 用户可以使用安抚一下。
- 用户可以记录一件小事。
- 用户可以在 App 内投喂桃桃。
- 用户可以看到小窝反馈。

安全边界：

- 用户侧不出现问答工具感、神秘信号感或诊断关系感表达。
- 桃桃不判断哪一方正确。
- 桃桃不诱导监控对方。
- 投喂不在负面情绪场景触发。
- 上传照片可删除。
- 共同回忆有双方可见边界。

设计确认：

- 页面符合轻未来、温暖、通透、拟人化方向。
- 桃桃是主视觉。
- 中间导航是聊天。
- 首页行动模块在共同回忆之前。
- 小窝不是商城。

## 推荐开发顺序

1. 可交互页面。
2. 账号与情侣绑定。
3. 桃桃生成流程。
4. 首页。
5. 聊天基础能力。
6. 吃什么。
7. 好好说。
8. 送礼灵感与安抚一下。
9. 记一件小事。
10. 投喂与小窝。
11. 小程序适配。
