# Stitch 高保真 UI 设计提示词

## 使用方式

把本文件作为 Stitch 的 UI 设计输入。建议先让 Stitch 生成移动端 App 首页、聊天页、小窝页、回忆页和唤醒桃桃流程，再基于同一视觉系统生成小程序版本。

## 总体设计 Brief

设计一个中国本土情侣 App，核心是两个人共同养成的拟人化小伙伴 `桃桃`。桃桃由用户上传的照片唤醒，并被生成成 3D 动态小生命。产品不是 AI 助手、不是关系评理器、不是纪念日工具，而是一个会聊天、会记事、会撒娇、会陪情侣做日常决定的关系小伙伴。

视觉方向：轻未来、温暖、通透、拟人化。使用发光白、浅蓝、桃粉、珊瑚红作为主调，整体干净、有呼吸感、有消费级 App 质感。不要做成深色科技风，不要做成金融 App，不要做成 SaaS 仪表盘。

用户第一眼应该看到桃桃，并马上看到 `今天我们可以一起` 的行动入口。共同回忆必须放在行动模块之后。

## 禁止文案

不要在 UI 中出现：

- AI 心灵信号
- 心灵共振
- 能量同步
- 智能助手
- AI 助手
- 机器人
- 问问
- 评理
- 裁决
- 关系分析

## 推荐文案

- 桃桃今天很开心，想和你们一起决定晚饭。
- 桃桃帮你换个温柔说法。
- 桃桃记住了这件小事。
- 桃桃想和你们一起做点什么。
- 要不要投喂桃桃一点小零花。
- 不投喂也没关系，桃桃还是会陪你们。

## 设计系统方向

### 色彩

- Base：#F7FAFF，发光白。
- Primary：#6CB9FF，温柔浅蓝。
- Accent：#FF7C73，桃粉珊瑚。
- Soft Accent：#FFD6E5，淡粉。
- Success：#8FD8A8，嫩绿。
- Text Primary：#1E3555。
- Text Secondary：#6F7F95。

### 字体

- 中文使用系统无衬线字体，偏圆润但不要幼稚。
- 正文 14-16px。
- 标题 20-28px。
- 重要数字可 28-34px。

### 圆角

- 卡片圆角 20-28px。
- 图标容器圆角 18-24px。
- 底部导航圆角可更柔和。

### 图标

使用 3D 软质图标或轻拟物图标，风格统一。不要使用生硬线框图标作为主要动作入口。

### 桃桃形象

桃桃必须是一个可爱的 3D 动态小生命，不是机器人、不是光球、不是抽象信号。它可以有柔软材质、大眼睛、轻微发光底座、简单肢体动作和可爱的表情。

## 页面 1：首页

生成 390x844 移动端高保真首页。

结构：

1. 顶部：双方头像、爱心连接、`我们在一起 520 天`、通知入口。
2. 主视觉：大尺寸 3D 桃桃，桃桃站在轻未来小空间里。
3. 桃桃气泡：`桃桃今天很开心，想和你们一起决定晚饭。`
4. 辅助状态：`桃桃的今日状态`，展示心情、活力、小愿望。
5. 零花钱罐：轻量显示，不要像支付入口。
6. 第一行动模块：`今天我们可以一起`，不要做宫格。桃桃先递出一个主建议，例如 `今晚先定吃什么`，下面只给 2-3 个轻选项，例如 `好好说`、`记小事`、`约会灵感`。
7. 第二模块：`共同回忆`，展示 3-4 张最近回忆卡。
8. 底部导航：`首页 / 回忆 / 桃桃 / 小窝 / 我们`，中间 `桃桃` 突出桃桃头像，代表三方群聊现场。

重点：

- 行动模块必须在共同回忆之前。
- 页面不要拥挤。
- 桃桃要有存在感。
- 不要出现 AI 术语。

## 页面 2：唤醒桃桃流程

生成 3-4 个关键步骤的移动端 UI。

步骤：

1. 上传照片：`上传一张你们都有感觉的照片，桃桃会从这里醒来。`
2. 确认对象：`桃桃想从这个小东西里醒来，可以吗？`
3. 选择性格：可爱、温柔、活泼、黏人、小太阳。
4. 预览桃桃：展示 3D 动态桃桃，按钮 `就唤醒这个桃桃`。

设计重点：

- 不要像头像生成器。
- 要像一个小生命被唤醒。
- 流程短，适合小程序。

## 页面 3：聊天

生成聊天页高保真 UI。

聊天页必须是 `情侣双方 + 桃桃` 的三方群聊，不是单人和桃桃对话。界面里要能看到她、他、桃桃三类消息，桃桃作为共同小伙伴参与两个人的决定、表达和记录。

结构：

- 顶部：她、他、桃桃三方头像，桃桃状态 `正在听你们说话`。
- 聊天区：她、他、桃桃三方消息。
- 轻入口：吃什么、好好说、记小事。不要把聊天页做成工具菜单。
- 输入框：`在三方群聊里说点什么`

交互感：

- 桃桃不是客服。
- 回复要短、温暖、有一点撒娇。
- 可生成行动卡片。

## 页面 4：吃什么

生成晚饭决策流程 UI。

流程：

1. 桃桃询问：`今天想轻松一点，还是吃点好吃的？`
2. 用户选择预算、口味、外卖/堂食/下厨。
3. 桃桃给 3 个建议。
4. 双方选择后确认。
5. 完成后可记录回忆或投喂桃桃。

重点：

- 不要做成复杂筛选器。
- 要像桃桃陪两个人商量。

## 页面 5：小窝

生成小窝页面。

结构：

- 桃桃在小窝中央。
- 小窝背景。
- 零花钱罐。
- 最近收到的礼物。
- 可更换装扮。
- 桃桃收到过的小开心。

重点：

- 小窝不是商城。
- 投喂入口温柔，不强迫。
- 付费反馈要可见：小窝变暖、桃桃开心、出现小礼物。

## 页面 6：共同回忆

生成回忆页。

结构：

- 回忆时间线。
- 图片卡片。
- 桃桃一句话。
- 来源场景。
- 双方确认状态。

重点：

- 回忆页可以更安静。
- 首页不要让回忆抢行动入口。

## 页面 7：我们

生成“我们的小窝”页面，不要生成后台式资料设置页。

内容：

- 首屏：今天一起做一件小事、在一起天数、共同回忆、小窝暖光、最近留下的事、桃桃收到过的小开心。
- 二级入口：桃桃的样子、我们的小约定、少打扰一点、边界与安心。
- 边界能力：相册与相机、小事提醒、来源照片可清掉、共同内容需要双方再次确认。

设计重点：

- 生活化、可信任，不像设置后台。
- 不做复杂关系测评。

## 小程序适配

小程序首发可以比 App 更轻：

- 首页。
- 聊天。
- 唤醒桃桃。
- 吃什么。
- 投喂。
- 小窝轻量页。

小程序需要更短的流程、更少的层级、更明确的按钮。
